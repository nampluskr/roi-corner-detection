# Line Intersection (line)

## 초록

Line Intersection은 "사각형의 코너는 네 변이 만나는 점"이라는 기하학적 사실을
파이프라인으로 옮긴 방법론이다. 이미지에서 직선 세그먼트들을 검출하고, 이를
사각형의 4개 변으로 그룹화한 뒤, 이웃한 변끼리의 교점 4개를 코너로 계산한다.
직선 검출기는 경량 학습형 검출기(M-LSD)를 기본으로 하며, 비학습 대안(Canny +
Hough)으로 교체할 수도 있다. 코너 자체는 한 점의 국소 증거뿐이지만 변은 이미지를
길게 가로지르는 수백 픽셀의 증거를 가지므로, 변을 먼저 찾고 코너를 유도하는 이
접근은 코너 근방이 반사나 잡음으로 훼손된 경우에도 강건하다. 교점은 직선 적합의
연속 해이므로 픽셀 격자에 묶이지 않는 정밀도를 가진다.

## 1. 서론

### 1.1 배경: 점보다 선이 증거가 많다

코너는 기하학적으로 두 직선의 교점이며, 이미지에서 코너 자체가 차지하는 영역은
몇 픽셀에 불과하다. 만약 그 몇 픽셀이 글레어(반사광)나 잡음으로 훼손되면, 코너를
직접 찾는 접근은 근거를 잃는다. 반면 사각형의 변은 이미지를 길게 가로지르는
연속된 경계로, 일부 구간이 훼손되어도 나머지 구간의 증거로 직선을 적합할 수
있다. 직선은 파라미터가 2개뿐인 강한 기하 모델이므로, 많은 관측(엣지 픽셀,
선분들)을 소수 파라미터에 적합하는 과정에서 국소 오차가 평균화된다. 즉 "변을
먼저, 코너는 유도로"라는 순서는 증거의 양과 모델의 강건성 측면에서 자연스러운
선택이다.

또한 물체가 화면의 대부분을 차지하는 조건에서는 네 변이 모두 길게 나타나므로
직선 검출이 특히 안정적이다. 이 방법론이 본 프로젝트 도메인(직선 경계가 뚜렷한
평면 패널)에 적합하다고 기대되는 이유이다.

### 1.2 문제 정의

입력은 컬러 이미지 한 장이고, 중간 출력은 직선 세그먼트들의 목록, 최종 출력은
TL(좌상단) $\to$ TR(우상단) $\to$ BR(우하단) $\to$ BL(좌하단) 시계 방향 순서의 코너 4개
좌표(정규화 [0, 1])이다. 파이프라인은 "직선 세그먼트 검출(학습형 또는 비학습형)"
과 "그룹화 + 교점 계산(비학습 기하 연산)"의 2단계이다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

**동차 좌표(homogeneous coordinates).** 직선과 점의 관계를 대수적으로 다루기
위해 2차원 점 $(x, y)$를 3차원 벡터 $\mathbf{p} = (x, y, 1)^\top$로 표기한다.
직선 $ax + by + c = 0$은 벡터 $\mathbf{l} = (a, b, c)^\top$로 표현되며, 점이
직선 위에 있을 조건은 내적으로 쓰인다.

$$
\mathbf{l}^\top \mathbf{p} = a x + b y + c = 0
$$

이 표기의 힘은 다음 두 쌍대 관계에 있다. 두 점 $\mathbf{p}_1, \mathbf{p}_2$를
지나는 직선과, 두 직선 $\mathbf{l}_1, \mathbf{l}_2$의 교점이 모두 외적(cross
product)으로 계산된다.

$$
\mathbf{l} = \mathbf{p}_1 \times \mathbf{p}_2, \qquad
\mathbf{p} = \mathbf{l}_1 \times \mathbf{l}_2
$$

외적의 성분은 다음과 같다.

$$
\mathbf{l}_1 \times \mathbf{l}_2 =
\begin{pmatrix}
b_1 c_2 - b_2 c_1 \\
a_2 c_1 - a_1 c_2 \\
a_1 b_2 - a_2 b_1
\end{pmatrix}
$$

교점의 2차원 좌표는 세 번째 성분으로 나누어 얻는다:
$(x, y) = (p_1 / p_3,\ p_2 / p_3)$. $p_3 = a_1 b_2 - a_2 b_1$이 0에 가까우면 두
직선이 평행하여 교점이 무한원점으로 발산하므로, $|p_3|$가 교점 계산의 안정성
지표가 된다.

**문제의 분해.** 사각형은 네 직선 $\mathbf{l}_{top}, \mathbf{l}_{right},
\mathbf{l}_{bottom}, \mathbf{l}_{left}$로 완전히 결정되고, 코너는 이웃 변의
교점이다.

$$
\hat{Y}_{TL} = \mathbf{l}_{left} \times \mathbf{l}_{top}, \quad
\hat{Y}_{TR} = \mathbf{l}_{top} \times \mathbf{l}_{right}, \quad
\hat{Y}_{BR} = \mathbf{l}_{right} \times \mathbf{l}_{bottom}, \quad
\hat{Y}_{BL} = \mathbf{l}_{bottom} \times \mathbf{l}_{left}
$$

따라서 문제는 "이미지 $\to$ 직선 세그먼트 집합 $\to$ 4개 변 직선 $\to$ 교점 4개"로
분해된다. 첫 변환만 학습 대상이고 나머지는 결정적 기하 연산이다.

### 2.2 모델 구조

직선 세그먼트 검출기는 학습형(M-LSD)을 기본으로 한다.

**(1) M-LSD의 세그먼트 표현.** M-LSD(Mobile Line Segment Detector)는 실시간
동작을 목표로 설계된 경량 직선 검출 신경망이다. 핵심은 선분을 "중심점 + 변위
벡터"로 표현하는 것이다. 선분의 두 끝점을 $\mathbf{s}, \mathbf{e}$라 하면 중심
$\mathbf{c} = (\mathbf{s} + \mathbf{e}) / 2$와 변위 $\mathbf{d} = (\mathbf{e} -
\mathbf{s}) / 2$로 등가 표현되고, 끝점은 $\mathbf{c} \pm \mathbf{d}$로 복원된다.
점 하나(중심)와 벡터 하나(변위)를 예측하는 문제로 바꾸면, keypoint 검출과 같은
지도(map) 예측 구조를 그대로 쓸 수 있다.

**(2) 출력 지도.** 인코더-디코더 형태의 경량 backbone(MobileNet 계열, 깊이별
분리 합성곱 기반)이 입력 이미지에서 공간 지도들을 예측한다.

- **중심 지도** $C \in [0, 1]^{H' \times W'}$: 각 위치가 어떤 선분의 중심일 확률.
- **변위 지도** $D \in \mathbb{R}^{4 \times H' \times W'}$: 각 위치가 중심일 때
  두 끝점으로의 변위 $(d_x, d_y, -d_x, -d_y)$.
- (보조) 끝점/길이/각도 지도: 학습을 돕는 부가 예측.

**전체 shape 흐름** (입력 $512 \times 512$, 출력 stride 2, 배치 크기 $N$):

| 단계 | 연산 | 출력 shape | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, 512, 512) | 컬러 이미지 배치 |
| 경량 backbone | 분리 합성곱 인코더-디코더 | (N, 64, 256, 256) | 공유 특징 |
| 중심 head | 1x1 Conv + sigmoid | (N, 1, 256, 256) | 선분 중심 확률 지도 |
| 변위 head | 1x1 Conv | (N, 4, 256, 256) | 끝점 변위 지도 |

```text
이미지 (3x512x512)
   $\to$ [경량 인코더-디코더]     $\to$ 공유 특징 (64x256x256)
   $\to$ [중심 head]              $\to$ 중심 지도 C (1x256x256)
   $\to$ [변위 head]              $\to$ 변위 지도 D (4x256x256)
   $\to$ [중심 국소 최대 + 변위 복원]      $\to$ 선분 목록 {(s_m, e_m)}
   $\to$ [4변 그룹화 + 직선 적합 + 교점]   $\to$ 코너 (4x2)
```

**세그먼트 디코딩.** 중심 지도에서 국소 최대(임계값 이상)를 뽑고, 그 위치의
변위 예측으로 끝점 쌍을 복원하면 선분 목록 $\{(\mathbf{s}_m, \mathbf{e}_m)\}$을
얻는다.

**공개 가중치의 채널 배치와 입력 규약 (구현 확정).** 본 구현이 사용하는 M-LSD
공개 가중치(`mlsd_large_512_fp32.pth`)는 16채널 tp-map을 출력하며, 채널 의미는
공개 저장소마다 표기가 달라 실측(정답 코너 대비 채널 조합 전수 비교)으로 확정했다:
채널 [0]이 중심 확률 logit, 채널 [8:12]가 시작/끝점 변위이고 나머지는 보조 채널이다.
또한 이 가중치는 $512 \times 512$ 입력과 $[-1, 1]$ 정규화, RGB에 alpha=1을 붙인
4채널 입력을 전제한다. 공유 데이터 파이프라인(224, ImageNet 정규화)과의 차이는
`LineModel.forward` 진입부에서 역정규화 $\to$ 512 업샘플 $\to$ $[-1, 1]$ 변환 $\to$
alpha 결합으로 흡수하며, 입력 크기를 224로 두면 성능이 크게 하락한다(실측 SR
0.94 $\to$ 0.68).

**(3) 비학습 대안.** 학습이 불가능하거나 불필요한 환경에서는 Canny 엣지 검출 +
확률적 Hough 변환으로 같은 형식의 선분 목록을 얻을 수 있다. 직선의 $(\rho,
\theta)$ 매개화($\rho = x\cos\theta + y\sin\theta$)에 대한 엣지 픽셀 투표로
선분을 찾는 고전 기법이다. 이후 단계(2.4절)는 선분 목록만 받으므로 검출기 교체가
파이프라인에 투명하다.

### 2.3 손실 함수

학습형 검출기(M-LSD 계열)의 손실은 출력 지도별 항의 합이다.

**(1) 중심 지도 손실.** 정답 선분들의 중심 위치에 가우시안 봉우리를 그린 타깃
$C^*$에 대해, 픽셀별 이진 교차 엔트로피(BCE)를 적용한다. 중심은 극소수
픽셀이므로 양성-음성 불균형이 심하고, 잘 맞힌 배경 픽셀의 손실을 자동 억제하는
focal 변형을 쓴다.

$$
\mathcal{L}_{center} = -\frac{1}{H'W'} \sum_{i,j}
(1 - p_{t})^{\gamma} \log p_{t}, \qquad
p_t = \begin{cases} \hat{C}_{ij} & C^*_{ij} = 1 \\ 1 - \hat{C}_{ij} & \text{그 외} \end{cases}
$$

$\gamma > 0$이 클수록 쉬운(이미 잘 맞힌) 픽셀의 그래디언트가 강하게 억제된다.

**(2) 변위 회귀 손실.** 정답 중심 위치 집합 $\mathcal{P}$에서만 변위 벡터를
SmoothL1로 회귀한다. 오차 $e$에 대해 SmoothL1은 $|e| < \beta$에서
$e^2 / 2\beta$, 그 외에서 $|e| - \beta/2$인 강건 손실이다.

$$
\mathcal{L}_{disp} = \frac{1}{|\mathcal{P}|} \sum_{(i,j) \in \mathcal{P}}
\sum_{t \in \{d_x, d_y\}} \mathrm{SmoothL1}\big(\hat{D}_t(i,j) - D^*_t(i,j)\big)
$$

**(3) 전체 손실.**

$$
\mathcal{L} = \mathcal{L}_{center} + \lambda_{disp} \, \mathcal{L}_{disp}
$$

**타깃 생성.** 코너 라벨만 있는 데이터에서 직선 타깃은 자동 유도된다: 이웃 코너
쌍을 잇는 4개 변이 곧 정답 선분이며, 긴 변은 일정 길이로 분할해 여러 중심-변위
타깃으로 나눌 수 있다. 즉 코너 CSV로부터 직선 감독이 무비용으로 생성된다.

사전학습된 공개 가중치(일반 장면의 선분 검출)를 그대로 쓰고 파인튜닝을 생략하는
운용도 가능하다. 이 경우 학습되는 파라미터가 없으므로 손실 함수는 해당 없음이
되고, 성능은 후처리 파라미터 튜닝으로만 조정한다.

### 2.4 후처리 이론

선분 목록 $\{(\mathbf{s}_m, \mathbf{e}_m)\}$에서 코너 4개를 계산하는 결정적
절차이다.

**1단계: 필터링.** 길이 $\|\mathbf{e}_m - \mathbf{s}_m\|$가 임계값 미만이거나
확신도가 낮은 선분을 제거한다. 짧은 선분은 방향 추정 분산이 커서 그룹화를
오염시킨다. 중심 확신도 상위 선분만 남기는 top-k 컷도 함께 적용하며, 이때 k는
고정하지 않고 2-pass로 시도한다: 먼저 엄격한 상위 50개로 4변 그룹화를 시도하고,
실패하면(변 미확보 등) 완화된 상위 200개로 재시도한다(구현 확정). 엄격한 값이
배경 잡음 선분을 배제해 대부분의 경우 더 정확하고, 완화된 값은 선분이 적은
어려운 샘플에서 실패 대신 근사 성공을 택하는 안전망이다.

**2단계: 방향 그룹화.** 선분의 방향각 $\alpha_m = \mathrm{atan2}(e_y - s_y,\
e_x - s_x)$은 $\alpha$와 $\alpha + \pi$가 같은 직선 방향이므로, 각도를 두 배로
접은 단위 벡터

$$
\mathbf{u}_m = (\cos 2\alpha_m,\ \sin 2\alpha_m)
$$

로 표현하면 주기성 문제가 사라진다. $\{\mathbf{u}_m\}$을 2개 군집으로 나누면
(k-means 등) "가로 계열"과 "세로 계열" 두 방향군을 얻는다. 원근 왜곡이 있어도
마주 보는 변끼리는 방향이 비슷하고 이웃 변과는 크게 다르므로(볼록 사각형 전제)
이 분리는 안정적이다.

**3단계: 변 분리.** 각 방향군 안에서 선분들을 두 변(마주 보는 쌍)으로 나눈다.
전체 선분들의 무게중심 $\mathbf{g}$를 기준으로, 각 선분 중심 $\mathbf{c}_m$의
부호 있는 거리(방향군의 공통 법선 $\mathbf{n}$ 방향으로의 사영)

$$
\delta_m = \mathbf{n}^\top (\mathbf{c}_m - \mathbf{g})
$$

의 부호로 두 그룹(예: 위쪽 변/아래쪽 변)을 가른다.

**4단계: 변별 직선 적합.** 각 변 그룹의 선분 끝점들을 모아 총최소제곱(total
least squares)으로 직선을 적합한다. 점들의 평균을 $\bar{\mathbf{p}}$, 공분산
행렬을

$$
\Sigma = \frac{1}{n} \sum_{i} (\mathbf{p}_i - \bar{\mathbf{p}})
(\mathbf{p}_i - \bar{\mathbf{p}})^\top
$$

라 하면, 직선 방향은 $\Sigma$의 최대 고유값의 고유벡터, 법선 $\mathbf{n} = (a,
b)$는 최소 고유값의 고유벡터이고, $c = -\mathbf{n}^\top \bar{\mathbf{p}}$이다.
이 적합은 점과 직선의 수직 거리 제곱합을 최소화하므로 축 방향 편향이 없다. 긴
선분에 더 큰 가중치(길이 비례)를 주면 짧은 잡음 선분의 영향이 줄어든다.

**5단계: 교점 계산과 정렬.** 이웃한 변끼리 외적으로 교점을 구하고(2.1절),
$|p_3|$가 임계값 미만(평행)이면 실패 처리한다. 네 교점을 무게중심 기준 각도로
시계 방향 정렬하고 TL부터 시작하도록 순환 이동한 뒤, [0, 1] 정규화와 볼록성/퇴화
검증을 수행한다.

**실패 모드.** (i) 선분 부족(4변 미확보), (ii) 방향군/변 분리 실패, (iii) 인접
변 평행으로 교점 발산, (iv) 교점의 화면 밖 이탈 또는 퇴화. 어느 경우든 성공
플래그를 False로 반환한다.

**정밀도 논의.** 최종 좌표는 다수 선분의 직선 적합이 만드는 연속 해의 교점이므로
픽셀 격자에 묶이지 않는다. 직선 파라미터의 추정 분산이 관측(선분, 엣지 픽셀)
수에 반비례해 줄어들므로, 변이 길게 보이는 조건일수록 교점 정밀도가 높아진다.

## 3. 결론

Line Intersection은 사각형의 기하학적 정의(네 변의 교점)를 그대로 알고리즘으로
만든 방법론이다. 장점은 다음과 같다.

- **증거의 집적**: 코너 한 점이 아니라 변 전체의 수백 픽셀 증거로 직선을
  적합하므로, 코너 근방의 국소 훼손(글레어 등)에 강건하다.
- **연속 좌표**: 직선 적합과 교점 계산이 모두 연속 해이므로 픽셀 격자 제약이
  없다.
- **경량성**: 학습형 검출기도 모바일 지향 경량 설계이고, 비학습 대안으로의
  교체도 가능하다.

한계는 그룹화 휴리스틱의 취약성이다. 배경에 강한 직선(선반, 프레임 등)이 많으면
방향군 분리가 오염되고, 곡선 경계나 극단적 원근에서는 "네 직선" 모델 자체가
어긋난다. 필터링/그룹화 임계값이 도메인 튜닝을 요구하는 것도 학습 기반 방법론과
대비되는 비용이다.

이 방법론은 경계가 곧고 대비가 뚜렷하며 배경 직선이 적은 조건에서 가장 적합하고,
"기하 모델을 명시적으로 쓰는" 설계의 대표로서 종단간 학습 계열과의 비교 축을
제공한다.

## 부록 A. 데이터 인터페이스

- **preprocessor**: 표준 코너 $(N, 4, 2)$에서 이웃 코너 쌍을 이어 4개 변 선분을
  만들고, 이를 중심 지도 + 변위 지도 타깃으로 변환한다 (2.3절의 타깃 생성). 긴
  변은 일정 길이로 분할해 다중 중심 타깃으로 나눈다.
- **postprocessor**: 선분 목록에 대해 필터링 $\to$ 방향 그룹화 $\to$ 변 분리 $\to$ 직선
  적합 $\to$ 외적 교점 $\to$ 정렬/정규화(2.4절)를 수행해 $(N, 4, 2)$를 만든다. 직선
  교점 등 이 방법론 전용 기하 연산은 `src/models/line/` 폴더 내부에 두며, 다른
  방법론이 필요로 하는 시점에 `src/utils/`로 승격한다 (CLAUDE.md 4절).
- **실패 모드**: 4변 미확보, 교점 발산, 퇴화 판정(`is_invalid_corners`) 시 성공
  플래그를 False로 반환한다. Evaluator가 SR(Success Rate)을 집계하고 정확도
  메트릭은 성공 표본에만 적용한다 (README.md 4절).

## 부록 B. 학습 전략

M-LSD 공개 사전학습 가중치(일반 장면 선분 검출)에서 출발하여, 3단계 전략을
파인튜닝으로 적용한다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | 공개 M-LSD 가중치 로드 | 일반 선분 검출 능력 확보 (학습 없음) | - |
| 2. Domain Adaptation | SmartDoc/MIDV-2020 + 합성 fringe 패턴 | 사각형 4변 위주의 선분 분포에 적응 | 1e-4 |
| 3. Fine-tuning | 실측 데이터 (소량) | 최종 정밀도 확보 | 1e-5 |

**비학습 변형의 운용**: Canny + Hough 대안을 쓰는 경우 학습 단계가 없으며, Canny
임계값과 Hough 투표 임계/최소 선분 길이를 검증셋에서 격자 탐색으로 정한다. 학습형
과 비학습형의 후처리(2.4절)는 동일하므로, 두 변형의 성능 차이가 곧 검출기 품질
차이를 보여 준다.

**조밀 타깃 2단계 파인튜닝 (구현 확정).** 파인튜닝의 성패는 타깃 밀도가 갈랐다.
코너 라벨에서 유도한 희소 타깃(변당 8개의 1픽셀 중심/변위)으로 학습하면 사전학습이
만든 조밀한 선분 예측장이 훼손되어 1 epoch 만에 검증 IoU가 급락했다. 반면 타깃을
조밀하게 재설계하니(변을 따라 약 2px 간격 stamp, 가우시안 이웃 픽셀에도 해당 픽셀
기준으로 보정한 endpoint 변위 기록, 이미지당 감독 픽셀 약 2400개) 파인튜닝이 처음
으로 개선 방향으로 동작했다. 확정 레시피는 2단계다.

| 단계 | 학습 대상 | 학습률 (backbone / decoder) | 실측 valid IoU |
|---|---|---|---|
| 0. 사전학습 고정 | 없음 (후처리만) | - | 0.385 |
| 1. warmup | 디코더만 (인코더 동결) | - / 1e-4 | 0.52 (30 epoch) |
| 2. 전체 파인튜닝 | 인코더 + 디코더 | 1e-5 / 5e-5 (재설정) | 0.87+ (4 epoch에 도달, 상승 중) |

단계 2에서 인코더까지 도메인에 적응하면 검출기가 패널 경계선 위주로 발화하여, 내부
텍스트선 오염이 사라지고 후처리 정확도가 급상승한다(oracle 상한 0.95). 이 스케줄은
`LineWrapper`의 `warmup_epochs`(기본 30)로 구현되어 있어, `on_fit_start`에서 인코더를
동결하고 warmup 도달 시 `on_epoch_end`에서 자동 해제하며 학습률을 위 표의 단계 2
값으로 재설정한다. 즉 `python scripts/train.py --method line --max_epochs 60`처럼
warmup보다 긴 epoch로 한 번 실행하면 전체 레시피가 재현된다. 후처리는 strict(상위
50개 세그먼트) $\to$ 실패 시 loose(상위 200개) 2패스로 동작한다.

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 4개 자유 직선의 교점 표현은 임의 볼록 사각형과 정확히 일치. 근사 없음 |
| F2 (단일 객체, 50% 이상) | 4변이 화면을 길게 가로질러 선분 증거가 풍부하고 그룹화가 안정적. 이 방법론의 핵심 전제 |
| F3 (코너 경계 내부) | 교점이 [0, 1] 내부인지 검증하는 단계를 후처리에 포함. 변 자체는 화면 안에 온전히 보임 |
| F4 (실측 소량) | 공개 사전학습 가중치 재사용 + 코너 라벨에서 직선 타깃 자동 유도(부록 B)로 데이터 요구 최소화 |
| F5 (서브픽셀 정밀도) | 다수 선분의 직선 적합 + 연속 교점 해로 격자 제약 없는 정밀도 (2.4절) |
| F6 (CPU 배포) | 경량 검출기 + 결정적 기하 후처리로 F6 상위 후보. 비학습 변형은 더 경량 |
| F7 (조명/글레어 변동) | 변 전체 증거의 집적이 국소 글레어 훼손을 평균화. 광학 증강 병행 |
| F8 (가림 없음) | 4변이 모두 관측된다는 가정이 방향군 2 + 변 2 분리 휴리스틱의 근거 |
