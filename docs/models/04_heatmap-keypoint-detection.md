# Heatmap Keypoint Detection (heatmap)

## 초록

Heatmap Keypoint Detection은 코너 좌표를 숫자 쌍이 아니라 공간 확률 지도(heatmap)로
표현하여 학습하는 방법론이다. 코너마다 채널 하나씩, 총 4장의 heatmap을 예측하며,
각 heatmap은 "이 픽셀에 해당 코너가 있을 법한 정도"를 담는다. 학습 타깃은 정답
코너 위치를 중심으로 하는 2차원 가우시안 봉우리이고, 추론 시에는 예측 heatmap의
공간 기댓값을 취하는 soft-argmax로 좌표를 복원한다. 출력이 입력과 공간적으로
정렬된 지도이므로 위치 정보가 신경망 끝까지 보존되고, soft-argmax가 픽셀 격자
사이의 연속 좌표를 산출하므로 서브픽셀 정밀도를 태생적으로 지원한다. 인체 자세
추정(pose estimation) 분야에서 표준으로 자리 잡은 keypoint 검출 접근을 사각형
코너 검출에 적용한 것이다.

## 1. 서론

### 1.1 배경: 좌표 회귀의 표현 병목

신경망이 좌표를 직접 출력하려면, 합성곱이 만든 공간 특징 맵을 어느 지점에서
벡터로 요약해야 한다. 이 요약(전역 풀링)이 일어나는 순간 "어디에"라는 정보는 채널
강도의 조합에 간접적으로 부호화되고, 미세한 위치 정보는 평균에 희석된다. keypoint
검출 연구에서는 이 병목을 피하는 방법으로 출력 자체를 공간 지도로 바꾸는 접근이
정착되었다. 위치를 "숫자로 말하는" 대신 "지도 위에 표시하게" 하면, 합성곱의 공간
등변성(입력이 이동하면 출력도 그만큼 이동하는 성질)이 출력까지 유지되어 위치
학습이 쉬워진다.

### 1.2 확률 지도라는 출력 표현

heatmap은 각 픽셀에 "여기에 keypoint가 있을 법한 정도"를 담은 지도이다. 정답을
한 픽셀만 1인 지도로 만들면 신호가 너무 희소해 학습이 어렵기 때문에, 정답 위치를
중심으로 부드럽게 퍼지는 가우시안 봉우리를 타깃으로 쓴다. 이렇게 하면 정답
근처의 예측도 부분 점수를 받으므로 그래디언트가 풍부해지고, "정답에서 얼마나
가까운가"라는 거리 정보가 손실에 자연스럽게 반영된다.

코너가 4개이므로 heatmap도 4장을 예측하며, 채널 순서가 코너 순서(TL, TR, BR,
BL)를 고정한다. 즉 채널의 정체성이 코너의 정체성이므로, 검출된 점들을 코너에
매칭하는 별도 단계가 필요 없다.

### 1.3 문제 정의

입력은 컬러 이미지 한 장이고, 중간 출력은 코너별 heatmap 4장, 최종 출력은 TL ->
TR -> BR -> BL 시계 방향 순서의 코너 4개 좌표이다. 각 좌표는 이미지 크기로 나눈
[0, 1] 정규화 좌표로 표현한다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

입력 이미지를 $I \in \mathbb{R}^{3 \times H \times W}$, 정답 코너를 정규화 좌표
$Y_k = (x_k, y_k)$, $k = 1, \dots, 4$로 표기한다. heatmap은 계산량을 줄이기 위해
입력보다 낮은 해상도 $H' \times W'$ ($H' = H / r$, $r$은 출력 stride, 예: $r = 4$)
로 예측한다.

**타깃 heatmap.** 코너 $k$의 heatmap 격자 좌표를 $(u_k, v_k) = (W' x_k,\ H' y_k)$
라 하면, 타깃은 그 점을 중심으로 하는 2차원 가우시안이다.

$$
H_k^*(i, j) = \exp\!\left(
- \frac{(j - u_k)^2 + (i - v_k)^2}{2 \sigma^2}
\right), \qquad
\begin{aligned}
&i = 0, \dots, H' - 1 \\
&j = 0, \dots, W' - 1
\end{aligned}
$$

$\sigma$는 봉우리의 퍼짐 폭이다. 봉우리 값은 중심에서 1이고 거리에 따라 매끄럽게
감소한다.

**학습 목표.** 모델은 4채널 heatmap을 출력하는 함수이다.

$$
f_\theta: \mathbb{R}^{3 \times H \times W} \to \mathbb{R}^{4 \times H' \times W'}
$$

최종 좌표는 예측 heatmap $\hat{H}_k$에 결정적 디코딩(soft-argmax, 2.4절)을
적용해 얻는다. 요약하면 이 방법론은 "좌표 -> 지도"로 타깃을 바꿔 학습하고,
"지도 -> 좌표"의 역변환을 미분 가능한 기댓값 연산으로 수행한다.

### 2.2 모델 구조

구조는 해상도를 줄이며 문맥을 모으는 backbone과, 해상도를 되살리며 heatmap을
그리는 업샘플링 head의 직렬 연결이다. keypoint 검출에서 널리 쓰이는 단순
구성(backbone + deconvolution head)을 기준으로 설명한다.

**(1) Backbone.** 합성곱 신경망(예: ResNet18)이 이미지에서 특징 맵을 추출한다.
합성곱은 작은 필터와 국소 영역의 가중합이며, 풀링/stride로 해상도를 줄일수록 넓은
문맥이 요약된다. 입력 $512 \times 512$에서 최종 특징 맵은
$512 \times 16 \times 16$이다. 이 시점의 특징은 "코너가 어느 구역에 있는가"를
알지만 해상도가 거칠다.

**(2) 전치 합성곱(deconvolution) head.** 거친 특징 맵을 단계적으로 업샘플링한다.
전치 합성곱은 입력의 각 값을 학습 가능한 필터 모양으로 넓은 출력에 "찍어" 누적하는
연산으로, stride $s$일 때 해상도가 $s$배 커진다. 일반 합성곱이 여러 입력을 하나로
모으는 연산이라면, 전치 합성곱은 하나의 입력을 여러 출력으로 퍼뜨리는 역방향
연산이다. 이를 2배 업샘플링 3회로 쌓으면 $16 \to 32 \to 64 \to 128$로 해상도가
복원된다 ($r = 4$).

**(3) 출력 계층.** 마지막 특징 맵에 $1 \times 1$ 합성곱을 적용해 채널을 4로
줄이면 코너별 heatmap logits $Z \in \mathbb{R}^{4 \times H' \times W'}$가 된다.
타깃이 [0, 1] 범위이므로 원소별 sigmoid $\sigma(z) = 1/(1+e^{-z})$를 적용해
$\hat{H} = \sigma(Z)$로 범위를 맞추는 구성이 자연스럽다 (타깃을 회귀 값으로 보고
활성화 없이 MSE를 적용하는 변형도 통용된다).

**전체 shape 흐름** (ResNet18, 입력 $512 \times 512$, $r = 4$, 배치 크기 $N$):

| 단계 | 연산 | 출력 shape | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, 512, 512) | 컬러 이미지 배치 |
| Backbone | 합성곱 계층 다수 | (N, 512, 16, 16) | 거친 해상도의 문맥 특징 |
| Deconv 1 | 전치 합성곱 x2 | (N, 256, 32, 32) | 해상도 복원 1단계 |
| Deconv 2 | 전치 합성곱 x2 | (N, 256, 64, 64) | 해상도 복원 2단계 |
| Deconv 3 | 전치 합성곱 x2 | (N, 256, 128, 128) | 해상도 복원 3단계 |
| 출력 | 1x1 Conv (+sigmoid) | (N, 4, 128, 128) | 코너별 heatmap |

```text
이미지 (3x512x512)
   -> [Backbone: 합성곱]        -> 문맥 특징 (512x16x16)
   -> [Deconv x3: 업샘플링]     -> 복원 특징 (256x128x128)
   -> [1x1 Conv (+sigmoid)]     -> heatmap (4x128x128), 채널 = 코너
   -> [soft-argmax]             -> 정규화 좌표 (4x2)
```

**이 구조가 의미하는 것**: 위치 정보가 벡터로 요약되는 지점이 없다. 코너의 위치는
학습 내내 "지도 위의 봉우리 위치"로 표현되고, 신경망은 봉우리를 정확한 자리에
그리는 법을 배운다. 대가는 조밀한 출력 텐서의 메모리/계산 비용이며, heatmap
해상도 $H' \times W'$가 커질수록 정밀도와 비용이 함께 증가하는 트레이드오프가
있다.

### 2.3 손실 함수

예측 heatmap과 타깃 heatmap의 픽셀별 평균 제곱 오차(MSE)를 쓴다.

$$
\mathcal{L} = \frac{1}{4 H' W'} \sum_{k=1}^{4} \sum_{i=1}^{H'} \sum_{j=1}^{W'}
\Big( \hat{H}_k(i, j) - H_k^*(i, j) \Big)^2
$$

MSE의 원소별 그래디언트는 $2(\hat{H} - H^*)$로, 예측과 타깃의 차이에 비례하는
매끄러운 교정 신호이다. 타깃이 [0, 1] 범위의 유계 값이므로 제곱 손실의 이상치
폭주 문제가 없어 MSE가 안정적으로 동작한다.

**$\sigma$의 트레이드오프.** 타깃 가우시안의 폭 $\sigma$는 학습 난이도와 정밀도의
균형을 정한다.

- $\sigma$가 작으면: 봉우리가 날카로워 위치 정보는 정밀하지만, 양성 신호가 극소수
  픽셀에 몰려 학습 초기 그래디언트가 빈약하다. 극단적으로 한 픽셀 타깃은 배경
  상수 예측과 손실 차이가 거의 없어 학습이 정체된다.
- $\sigma$가 크면: 그래디언트는 풍부하지만 봉우리가 뭉툭해져, 최적해 근방에서
  봉우리 중심의 위치 구분력이 떨어진다.

경험적으로 heatmap 해상도의 1-2% 수준(예: $128$ 격자에서 $\sigma = 2$)이 출발점이
된다.

**배경 지배 문제.** heatmap의 대부분은 값이 0에 가까운 배경이다. 픽셀 평균 MSE는
배경 오차가 총합을 지배할 수 있으나, 가우시안 타깃이 봉우리 주변에 넓은 경사를
만들어 위치 신호를 보존하므로 실용적으로 문제가 완화된다. 필요 시 봉우리 근방
픽셀에 가중치를 두는 가중 MSE, 또는 잘 맞힌 픽셀의 손실을 줄이는 focal 계열
변형으로 배경 항을 더 억제할 수 있다.

**좌표 손실과의 결합.** 2.4절의 soft-argmax는 미분 가능하므로, 디코딩된 좌표에
직접 좌표 회귀 손실(L1 등)을 추가로 걸 수 있다 (integral regression). 이 경우
heatmap MSE가 표현을 안정화하고 좌표 손실이 최종 목표를 정렬하는 이중 감독이
된다.

$$
\mathcal{L}_{total} = \mathcal{L}_{MSE} + \lambda \, \frac{1}{8}
\sum_{k=1}^{4} \big\| \hat{Y}_k - Y_k \big\|_1
$$

### 2.4 후처리 이론

후처리는 heatmap 4장을 좌표 4쌍으로 바꾸는 결정적 디코딩이다.

**hard argmax와 양자화 오차.** 가장 단순한 디코딩은 최댓값 픽셀을 취하는 것이다.

$$
(\hat{v}_k, \hat{u}_k) = \arg\max_{(i, j)} \hat{H}_k(i, j)
$$

이 방식의 좌표는 정수 격자에 묶이므로 최대 0.5픽셀의 양자화 오차가 생기고, 입력
해상도로 환산하면 $0.5 r$픽셀(stride 4에서 2픽셀)이다. 서브픽셀 정밀도가 목표라면
이 오차는 허용할 수 없다.

**soft-argmax (공간 기댓값).** heatmap을 확률 분포로 정규화한 뒤 좌표의 기댓값을
취하면 연속 좌표를 얻는다. 먼저 온도 $\beta > 0$의 softmax로 heatmap을 공간 확률
분포로 바꾼다.

$$
p_k(i, j) = \frac{\exp\big(\beta \hat{H}_k(i, j)\big)}
{\sum_{i', j'} \exp\big(\beta \hat{H}_k(i', j')\big)}
$$

이 분포에서 좌표의 기댓값을 계산한다.

$$
\hat{u}_k = \sum_{i, j} j \; p_k(i, j), \qquad
\hat{v}_k = \sum_{i, j} i \; p_k(i, j)
$$

기댓값은 픽셀 인덱스의 실수 가중 평균이므로 격자 사이의 값을 자유롭게 가진다.
이것이 서브픽셀 정밀도의 원천이다. 또한 모든 연산(지수, 합, 곱)이 미분 가능하므로
좌표에 손실을 직접 걸 수 있다 (2.3절의 integral regression).

**온도 $\beta$의 역할.** $\beta \to \infty$이면 분포가 최댓값 픽셀에 집중되어
hard argmax로 수렴하고, $\beta$가 작으면 분포가 퍼져 기댓값이 heatmap 전역의
질량에 끌린다. 실무적 주의점은 다음과 같다.

- **다봉/배경 질량 편향**: 기댓값은 분포 전체의 무게중심이므로, 봉우리가 두 개
  생기거나 배경에 잔여 질량이 있으면 좌표가 봉우리들 사이 또는 지도 중앙 쪽으로
  끌린다. $\beta$를 충분히 크게 잡거나, argmax 주변의 국소 창 안에서만 기댓값을
  계산하는 변형으로 완화한다.
- **퇴화 heatmap**: 모델이 확신 없는 입력에서 거의 평평한 heatmap을 내면
  soft-argmax는 지도 중앙 근처를 반환한다. 최댓값의 크기를 확신도로 삼아 품질
  판단에 활용할 수 있다.

**정규화.** 격자 좌표를 heatmap 크기로 나누어 [0, 1] 정규화 좌표로 변환한다.

$$
\hat{x}_k = \frac{\hat{u}_k}{W'}, \qquad \hat{y}_k = \frac{\hat{v}_k}{H'}
$$

채널 순서가 코너 순서이므로 4채널의 결과를 그대로 쌓으면 표준 코너 $(4, 2)$가
완성된다.

## 3. 결론

Heatmap Keypoint Detection은 "위치는 지도 위에 표시하는 것이 가장 배우기 쉽다"는
keypoint 검출의 표준 통찰을 코너 검출에 적용한 방법론이다. 장점은 다음과 같다.

- **공간 정보 보존**: 위치가 벡터로 요약되는 병목이 없어, 합성곱의 공간 등변성이
  출력까지 유지된다.
- **태생적 서브픽셀**: soft-argmax의 기댓값 디코딩이 격자 사이의 연속 좌표를
  산출한다.
- **풍부한 학습 신호**: 가우시안 타깃이 정답 근방 예측에 부분 점수를 주어
  그래디언트가 조밀하다.
- **순서 자동 결정**: 채널 정체성이 코너 정체성이므로 매칭/정렬 단계가 없다.

한계는 해상도-비용 트레이드오프와 디코딩의 분포 민감성이다. heatmap 해상도가
정밀도의 상한을 정하므로 고정밀에는 큰 출력 텐서가 필요하고, soft-argmax는 다봉
분포나 배경 잔여 질량에 좌표가 끌리는 편향이 있어 온도와 국소 창 설계가 필요하다.

이 방법론은 서브픽셀 정밀도가 최우선 요구인 조건에서 이론적으로 가장 유리한
선택지이며, 전역 회귀/국소 검출 계열과의 비교에서 정밀도 상한을 가늠하는 기준을
제공한다.

## 부록 A. 데이터 인터페이스

- **preprocessor**: 표준 코너 $(N, 4, 2)$를 출력 해상도의 가우시안 heatmap
  $(N, 4, H', W')$로 변환한다 (2.1절의 $H_k^*$). 정규화 좌표를 heatmap 격자
  좌표로 스케일한 뒤 채널별로 가우시안 봉우리를 그린다.
- **postprocessor**: raw 출력 $(N, 4, H', W')$에 soft-argmax(2.4절)를 적용하고
  heatmap 크기로 나누어 $(N, 4, 2)$ 정규화 좌표를 만든다. 채널 순서가 코너
  순서이므로 정렬이 없다.
- **실패 모드**: 기댓값 디코딩은 항상 4쌍의 좌표를 반환하므로 "미검출" 실패가
  구조적으로 없다. 다만 퇴화 판정(`is_invalid_corners`)은 동일하게 적용하며,
  평평한 heatmap이 만드는 중앙 편향 좌표는 이 검증에서 걸러질 수 있다.

## 부록 B. 학습 전략

3단계 학습 전략을 적용한다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | SmartDoc + MIDV-2020 (공개) | 일반적인 코너 heatmap 예측 능력 학습 | 1e-3 |
| 2. Domain Adaptation | 합성 fringe 패턴 | 검사 도메인의 화면 특성에 적응 | 1e-4 |
| 3. Fine-tuning | 실측 데이터 (소량) | 최종 서브픽셀 정밀도 확보 | 1e-5 |

**타깃 자동 생성**: heatmap 타깃은 preprocessor가 코너 CSV로부터 매번 생성하므로
어느 단계의 데이터든 코너 라벨만 있으면 사용 가능하다. $\sigma$는 전 단계에서
동일하게 유지해 타깃 분포의 일관성을 지킨다.

**backbone 초기화**: backbone은 ImageNet 사전학습 가중치, deconv head는 무작위
초기화로 시작한다.

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 코너별 독립 heatmap이라 4점이 임의 볼록 사각형을 자유롭게 표현. 형상 근사 없음 |
| F2 (단일 객체, 50% 이상) | 채널당 봉우리가 하나뿐인 조건이므로 soft-argmax의 다봉 편향 위험이 낮음 |
| F3 (코너 경계 내부) | 기댓값 좌표는 항상 heatmap 격자 내부이므로 [0, 1]을 벗어나지 않음. 가우시안 봉우리도 잘리지 않고 온전히 그려짐 |
| F4 (실측 소량) | 3단계 학습 전략(부록 B). 조밀한 heatmap 감독이 소량 데이터 학습을 보강 |
| F5 (서브픽셀 정밀도) | soft-argmax의 연속 좌표로 태생적 지원 (2.4절). 본 프로젝트에서 F5 최상위 후보 |
| F6 (CPU 배포) | deconv head와 조밀 출력으로 직접 회귀보다 무거움. heatmap 해상도(stride r)로 비용-정밀도 조절 |
| F7 (조명/글레어 변동) | 광학 증강(ColorJitter, GaussianBlur, GaussianNoise) 적용. heatmap 타깃은 광학 변화에 불변 |
| F8 (가림 없음) | 코너가 항상 보이므로 채널별 봉우리 소실이 없다는 가정이 성립. 확신도 기반 예외 처리 단순화 |
