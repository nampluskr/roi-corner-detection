# Torchvision Detection (torchdet)

## 초록

Torchvision Detection은 4개 코너를 각각 "코너를 중심으로 하는 소형 박스 객체"로 보고
객체 검출 프레임워크로 그 박스들을 찾은 뒤 박스 중심을 코너로 삼는 방법론이며, 검출기를
직접 설계하는 대신 성숙한 표준 검출기(Faster R-CNN, RetinaNet, SSD) 전체를 backbone,
특징 피라미드, 검출 head까지 통째로 재사용한다는 점이 특징이다. 박스의 클래스를 코너의
정체성(TL, TR, BR, BL)으로 정의하므로 검출과 동시에 순서가 결정되고, 클래스마다 정확히
하나의 인스턴스만 존재하는 전제에서 클래스별 최고 점수 박스 하나를 골라 그 중심을 코너로
디코딩한다. 앵커(anchor), 특징 피라미드(FPN), 영역 제안(RPN), 비최대 억제(NMS) 등
십수 년간 검증된 검출 요소를 그대로 이식해 강건성과 사전학습 전이를 얻는 대신, 검출기
전체가 무거워 배포 비용이 크고 학습 파이프라인이 복잡하다는 절충을 가진다.

## 1. 서론

### 1.1 배경: 성숙한 객체 검출 프레임워크를 통째로 재사용한다

객체 검출은 "무엇이 어디에 있는가"를 동시에 맞히는 문제로, 위치 추정(회귀)과 범주 판정
(분류)이 결합된 복합 과제이다. 이 분야는 표준 구조가 깊게 축적되어 있다. 두 단계
(two-stage) 검출기 Faster R-CNN은 영역 제안망(RPN)으로 후보를 추린 뒤 정밀 분류/회귀로
다듬고, 단일 단계(one-stage) 검출기 RetinaNet과 SSD는 격자 위에서 박스를 한 번에
예측한다. 여기에 다중 스케일 특징 피라미드(FPN), 앵커 기반 박스 파라미터화, 비최대
억제(NMS) 같은 공통 요소가 결합된다. 코너 검출을 "소형 박스 객체 검출"로 바꾸면 이
검증된 프레임워크를 그대로 활용할 수 있고, 대규모 검출 데이터로 사전학습된 가중치까지
이식할 수 있다.

직접 설계한 격자 검출기가 단일 스케일 head와 최소 후처리로 문제를 압축하는 것과 달리,
torchdet는 라이브러리 검출기의 다중 스케일, 앵커, 영역 제안, NMS 같은 성숙한 장치를
포기 없이 사용한다. 대가는 무게와 복잡도이다.

### 1.2 코너를 소형 박스 객체로 바꾸는 아이디어

코너는 점이므로 그 자체로는 박스가 아니다. 그러나 코너를 중심으로 하는 고정 크기의
가상 박스를 정의하면 "코너 4개 찾기"가 "작은 박스 객체 4개 검출하기"가 된다. 박스의
클래스를 코너 순번(TL, TR, BR, BL)으로 두는 것이 핵심으로, 검출된 박스의 중심이 코너
위치를, 클래스가 코너 정체성을 주므로 별도의 순서 정렬이 필요 없다. 사각형 전체를 박스
하나로 근사하는 것이 아니라 코너별 소형 박스를 두므로 원근 왜곡된 임의 볼록 사각형도
왜곡 없이 다룬다.

### 1.3 문제 정의

입력은 컬러 이미지 한 장이고, 최종 출력은 TL $\to$ TR $\to$ BR $\to$ BL 순서의 코너 4개 좌표
(이미지 크기로 나눈 [0, 1] 정규화)이다. 중간 표현은 검출기가 내는 박스 집합
(각 박스의 좌표, 클래스, 점수)이며, 후처리에서 클래스별로 가장 확신도 높은 박스 하나씩을
골라 그 중심을 코너로 변환한다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

입력 이미지를 $I \in \mathbb{R}^{3 \times H \times W}$, 정답 코너를
$Y_k = (x_k, y_k)$, $k = 1, \dots, 4$로 둔다. 코너 $k$를 중심으로 고정 기준 크기 $w_0$의
박스 $b_k = (x_k, y_k, w_0, h_0)$를 정의하고, 클래스 $c_k = k$를 부여한다. 학습 타깃은
이미지당 박스 4개와 그 클래스의 집합 $T = \{(b_k, c_k)\}_{k=1}^{4}$이다.

검출기는 이미지에서 임의 개수의 검출 결과를 내는 함수이다.

$$
f_\theta: \mathbb{R}^{3 \times H \times W} \to
\big\{ (\hat{b}_m, \hat{c}_m, \hat{s}_m) \big\}_{m=1}^{M}
$$

각 결과는 박스 $\hat{b}_m$, 클래스 $\hat{c}_m \in \{1, \dots, 4\}$, 점수
$\hat{s}_m \in [0, 1]$이다. 최종 코너는 학습 파라미터가 없는 후처리 $g$가 클래스별 최고
점수 박스의 중심에서 복원한다(2.4절).

$$
\hat{Y} = g\big(\{ (\hat{b}_m, \hat{c}_m, \hat{s}_m) \}\big) \in [0, 1]^{4 \times 2}
$$

### 2.2 모델 구조

torchvision 검출기는 공통적으로 backbone(특징 추출), 다중 스케일 특징, 앵커 기반 박스
예측 head로 구성된다. 공통 요소를 먼저 정리하고 세 계열을 본다.

**(1) 특징 피라미드(FPN).** 물체 크기는 다양하므로 한 스케일의 특징만으로는 부족하다.
FPN은 backbone의 여러 해상도 특징을 위에서 아래로 결합해, 작은 물체용 고해상도부터 큰
물체용 저해상도까지 여러 레벨의 특징 지도를 만든다. 각 레벨에서 같은 검출 head가
적용되므로, 다양한 크기의 코너 박스를 스케일별로 나눠 담당한다.

**(2) 앵커(anchor)와 박스 파라미터화.** 격자 각 위치에 여러 크기/종횡비의 기준 박스
(앵커)를 미리 깔고, 모델은 각 앵커에 대한 보정량과 물체 여부를 예측한다. 앵커
$(x_a, y_a, w_a, h_a)$에 대한 보정 $(t_x, t_y, t_w, t_h)$로 박스를 복원하는 표준식은
다음과 같다.

$$
\hat{x} = x_a + t_x w_a, \quad \hat{y} = y_a + t_y h_a, \quad
\hat{w} = w_a e^{t_w}, \quad \hat{h} = h_a e^{t_h}
$$

넓은 범위의 좌표 회귀를 "가까운 앵커 선택 + 작은 보정 회귀"로 분해하는 것이 앵커
방식의 핵심이며, 학습을 안정화한다. 앵커와 정답 박스는 IoU(교집합/합집합) 임계값으로
짝지어 양성/음성을 정한다.

**(3) 두 단계 검출기: Faster R-CNN.** 두 단계 구조는 검출을 "제안"과 "정밀화"로 나눈다.
1단계 영역 제안망(RPN)은 앵커마다 물체 여부와 대략적 박스를 예측해 후보 영역을 추린다.
2단계는 각 후보 영역의 특징을 RoIAlign으로 고정 크기로 잘라내(이중선형 보간으로 격자
정렬 오차를 줄인 풀링) 분류와 정밀 박스 회귀를 수행한다. 두 번 다듬으므로 정밀하지만
무겁다.

**(4) 단일 단계 검출기: RetinaNet.** 제안 단계 없이 FPN 각 레벨의 모든 앵커에서 직접
분류와 박스 회귀를 예측한다. 양성 앵커(4개 코너 부근)가 음성 앵커(대다수 배경)에 비해
극히 적은 불균형이 문제인데, RetinaNet은 focal loss로 이를 완화한다. 정답 클래스 확률
$p_t$에 대해

$$
\mathcal{L}_{\mathrm{focal}} = -(1 - p_t)^{\gamma} \log p_t
$$

로, 이미 잘 맞힌 다수 음성 앵커의 그래디언트를 $(1 - p_t)^{\gamma}$가 억제한다.

**(5) 단일 단계 검출기: SSD.** VGG 계열 backbone의 여러 해상도 특징 맵마다 서로 다른
크기의 기준 박스(default box)를 깔고, 각 위치에서 클래스 점수와 박스 보정을 예측한다.
다중 스케일 특징 맵 자체가 크기 다양성을 담당하며, 구조가 단순하고 빠르다.

**(6) 출력과 NMS.** 세 계열 모두 학습된 파라미터가 임의 개수의 후보 박스를 내고, 같은
물체에 겹치는 중복 박스를 비최대 억제(NMS)로 정리한다. NMS는 점수 순으로 박스를 보며,
이미 채택한 박스와 IoU가 임계값 이상으로 겹치는 후보를 제거한다. 추론 시 검출기는 점수
임계값과 NMS를 거친 최종 박스/클래스/점수 목록을 반환한다.

**전체 흐름** (배치 크기 $N$):

| 단계 | 연산 | 산출 | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, H, W) | 컬러 이미지 배치 |
| backbone + FPN | 합성곱 + 피라미드 결합 | 다중 스케일 특징 | 크기별 특징 지도 |
| 검출 head | (RPN+RoIAlign) 또는 dense 예측 | 앵커별 클래스/박스 | 후보 검출 |
| NMS/점수 임계 | 중복 억제 | 박스/라벨/점수 목록 | 최종 검출 결과 |
| 후처리 | 클래스별 top-1 중심 | (N, 4, 2) | 코너 좌표 |

```text
이미지 (3xHxW)
   $\to$ [backbone + FPN]        $\to$ 다중 스케일 특징
   $\to$ [검출 head + 앵커]      $\to$ 후보 박스/클래스/점수
   $\to$ [NMS + 점수 임계]       $\to$ 최종 검출 목록
   $\to$ [클래스별 top-1 중심]   $\to$ 코너 (N,4,2)
```

**직접 설계 격자 검출기와의 차이.** 코너용으로 직접 설계한 단일 격자 검출기가 하나의
해상도, 앵커 없는 셀별 예측, NMS 제거(클래스별 top-1)로 문제를 최소 구성하는 것과 달리,
torchdet는 다중 스케일 FPN, 앵커, (두 단계의 경우) 영역 제안, NMS 같은 성숙한 장치를
그대로 유지한 채 코너 도메인에 미세조정한다.

### 2.3 손실 함수

검출기 내부의 다항 손실을 그대로 사용한다. 계열별로 항 구성이 다르다.

- **Faster R-CNN**: RPN의 물체 여부 손실과 박스 회귀 손실 + 2단계 분류 손실과 박스 회귀
  손실의 합. 박스 회귀는 SmoothL1(작은 오차는 제곱, 큰 오차는 절댓값으로 이상치에 강건),
  분류는 교차 엔트로피이다.
- **RetinaNet**: 앵커별 분류에 focal loss(2.2절), 박스 회귀에 SmoothL1을 쓴다.
- **SSD**: 기준 박스별 분류(교차 엔트로피, 음성 표본을 점수 순으로 일부만 쓰는 hard
  negative mining 포함)와 박스 회귀(SmoothL1)의 합이다.

세 경우 모두 앵커-정답 짝짓기(IoU 기반 할당) 위에서 손실이 계산되며, 학습 시 검출기는
이미지와 타깃(박스, 클래스)을 받아 이 손실들의 합을 반환한다. 코너 박스는 이미지당 4개
뿐이라 양성 표본이 희소하므로, focal loss나 hard negative mining 같은 불균형 완화 장치가
그대로 유효하다.

### 2.4 후처리 이론

후처리 $g$는 검출 목록에서 코너 4개를 복원하는 결정적(비학습) 절차이다. 추론 시 검출기가
이미 점수 임계와 NMS를 적용하므로, 입력은 정리된 박스/클래스/점수 목록이다.

**1단계: 클래스별 top-1 선택.** 코너 $k$(클래스 $k$)에 대해 라벨이 $k$인 검출 중 점수가
최대인 박스 하나를 고른다.

$$
m_k^* = \arg\max_{m:\ \hat{c}_m = k} \hat{s}_m
$$

일반 검출은 한 클래스에 여러 인스턴스가 있을 수 있어 NMS 후에도 다수 박스가 남지만,
이 문제는 클래스마다 정확히 하나의 코너만 존재함이 보장되므로(가림 없음 전제) 클래스별
top-1 하나로 충분하다.

**2단계: 박스 중심 $\to$ 코너.** 선택된 박스 $\hat{b}_{m_k^*} = (x_1, y_1, x_2, y_2)$의
중심을 코너로 삼고 이미지 크기로 정규화한다.

$$
\hat{x}_k = \frac{x_1 + x_2}{2H_W}, \qquad \hat{y}_k = \frac{y_1 + y_2}{2H_H}
$$

여기서 $H_W, H_H$는 입력 폭과 높이이다. 클래스 순서가 곧 코너 순서(TL, TR, BR, BL)
이므로 별도 정렬이 없다.

**3단계: 유효성 검증.** 어떤 코너 클래스의 검출이 아예 없거나, top-1 점수가 임계값 미만
이거나, 4점이 퇴화(코너 간 거리 과소) 판정에 걸리면 검출 실패로 보고 성공 플래그를
False로 반환한다. 검출 실패라는 상태가 존재하므로 성공률(SR)을 함께 추적한다.

## 3. 결론

Torchvision Detection은 "코너 = 클래스가 정체성인 소형 박스 객체"라는 문제 변환으로 성숙한
객체 검출 프레임워크 전체를 코너 검출에 이식한 방법론이다. 장점은 다음과 같다.

- **검증된 검출 장치**: FPN, 앵커, 영역 제안, NMS 등 오랜 검증을 거친 요소를 포기 없이
  사용해 강건성이 높다.
- **사전학습 전이**: 대규모 검출 과제로 학습된 가중치가 물체 국소화 능력을 이미 가져,
  코너 도메인 데이터가 적어도 빠르게 적응한다.
- **순서 자동 결정**: 클래스가 코너 정체성을 주므로 정렬/매칭 후처리가 없고, 클래스별
  top-1이 NMS 이후 선택을 단순화한다.

한계는 무게와 복잡도이다. 다중 스케일과 (두 단계의) 영역 제안을 포함한 검출기는 배포
비용이 크고(F6 불리), 다항 손실과 앵커 짝짓기로 학습 파이프라인이 복잡하다. 또한 코너
박스가 이미지당 4개뿐이라 양성 표본이 희소해 데이터 요구가 크다. 이 방법론은 직접 설계한
경량 격자 검출기와 "라이브러리 재사용 대 직접 설계"라는 대조 축을 제공한다.

## 부록 A. 데이터 인터페이스

- **model**: `TorchdetModel`은 torchvision 검출 빌더(`fasterrcnn_resnet50_fpn`,
  `retinanet_resnet50_fpn`, `ssd300_vgg16`)를 `num_classes=5`(배경 + 코너 4개)로 생성한다.
  dataloader가 이미 ImageNet 정규화를 적용하므로 검출기 내부 정규화(`transform`의
  image_mean/std)는 항등으로 비활성화한다. forward는 학습 시 `(images, targets)`로 손실
  dict를, 추론 시 `(images)`로 이미지별 `{boxes, labels, scores}` dict 리스트를 반환한다.
- **preprocessor**: 표준 코너 $(N, 4, 2)$를 이미지별 검출 타깃 dict로 변환한다. 코너별로
  픽셀 중심에 고정 기준 크기의 소형 박스를 두되, 경계 근처에서 박스가 이미지를 벗어나지
  않도록 대칭 클리핑(최소 크기 보장)한 xyxy 박스와 클래스 라벨 $\{1, 2, 3, 4\}$
  (0=배경)를 담는다.
- **postprocessor**: 추론 검출 목록에서 클래스 $1..4$별 최고 점수 박스의 중심을 이미지
  크기로 정규화해 $(N, 4, 2)$를 만든다(2.4절). 클래스 누락, 점수 임계 미만, 퇴화
  판정(`is_invalid_corners`) 시 해당 표본을 NaN으로 반환한다.
- **실패 모드**: Evaluator의 `SuccessRate`가 유효 표본 비율로 SR을 집계하고, 정확도
  메트릭은 성공 표본에만 적용하며 항상 SR과 함께 보고한다 (README.md 4절).

## 부록 B. 학습 전략

3단계 학습 전략을 적용한다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | SmartDoc + MIDV-2020 (공개) | 코너 박스 검출의 일반 능력 확보 | 1e-3 |
| 2. Domain Adaptation | 합성 fringe 패턴 | 검사 도메인의 화면 특성에 적응 | 1e-4 |
| 3. Fine-tuning | 실측 데이터 (소량) | 최종 정밀도 확보 | 1e-5 |

**가중치 초기화**: backbone, 특징 피라미드, 영역 제안망 등은 대규모 공개 검출 과제로
사전학습된 가중치에서 시작하고, 클래스 수가 5로 바뀌는 분류 head(및 클래스별 박스 회귀
head)는 형태가 달라 이식 대상에서 제외되어 무작위 초기화된다. 사전학습된 국소화 능력
덕분에 코너 도메인 적응이 빠르다.

**합성 단계 의존성**: 검출 손실이 다항(분류/박스/제안)으로 구성되고 코너 박스가 이미지당
4개뿐이라 양성 학습 신호가 희소하므로, 데이터 요구가 상대적으로 크고 합성 데이터 단계의
비중이 큰 방법론이다 (F4).

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 코너별 소형 박스는 quad 전체를 box로 근사하는 것이 아니므로 저촉 없음. 4개 중심이 임의 볼록 사각형을 자유롭게 표현 |
| F2 (단일 객체, 50% 이상) | 코너 간 거리가 충분해 클래스별 top-1 선택이 안정적으로 동작하는 근거 |
| F3 (코너 경계 내부) | 코너 박스의 대칭 클리핑으로 박스가 항상 이미지 내부에 있고 중심은 코너에 고정됨 |
| F4 (실측 소량) | 대규모 검출 사전학습 가중치 이식 + 3단계 학습(부록 B). 손실 구조가 복합적이라 합성 단계 의존이 큼 |
| F5 (서브픽셀 정밀도) | 박스 중심 좌표는 연속값이나 특징 stride 해상도의 한계 존재. cornerSubPix 후정제 옵션 |
| F6 (CPU 배포) | 다중 스케일과 (두 단계의) 영역 제안을 포함한 검출기는 무거움. SSD/경량 backbone 계열이 상대적 대안 |
| F7 (조명/글레어 변동) | 광학 증강(ColorJitter, GaussianBlur, GaussianNoise) 적용. 앵커/박스 타깃은 광학 변화에 불변 |
| F8 (가림 없음) | 클래스별 정확히 1개 검출 가정(top-1)의 근거 |
