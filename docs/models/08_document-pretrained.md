# Document Pretrained (doc)

## 초록

Document Pretrained는 문서 이미지 도메인에서 대규모로 사전학습된 기하 인식 모델
(DocTr, DocScanner 계열)의 표현을 재사용하여, 소량의 데이터만으로 코너 검출에
적응시키는 방법론이다. 문서 스캔/펴기(rectification) 분야는 "평면 사각형이 원근
왜곡되어 찍힌 이미지에서 그 기하를 복원한다"는 본 과제와 동일한 기하 구조를
다루며, 공개된 사전학습 가중치가 존재한다. 이 방법론의 raw 출력과 후처리는 직접
회귀와 동일(좌표 logits 8개 $\to$ sigmoid $\to$ reshape)하고, 차별점은 모델 초기화와
학습 절차에 있다: 무작위 또는 ImageNet 초기화 대신 문서 기하를 이미 아는 가중치
에서 출발하여, 전이학습(transfer learning)과 파인튜닝(fine-tuning) 기법으로
표적 도메인에 few-shot 적응한다. 실측 데이터가 극히 적은 조건에 대한 대응책
성격의 방법론이다.

## 1. 서론

### 1.1 배경: 이미 배운 모델에서 시작하기

신경망 학습에 필요한 데이터 양은 배워야 할 것의 양에 비례한다. 무작위 초기화에서
시작하면 "이미지란 무엇인가"(엣지, 질감, 형상)부터 "사각형의 기하"까지 전부
데이터에서 배워야 하지만, 비슷한 문제를 이미 푼 모델에서 시작하면 마지막 간극만
배우면 된다. 이것이 전이학습의 기본 논리이다.

문서 이미지 처리 분야에는 이 목적에 특히 잘 맞는 사전학습 모델들이 있다. 문서
펴기(document rectification/unwarping) 모델은 카메라로 비스듬히 찍힌 문서
이미지에서 문서의 기하 구조(경계, 왜곡장)를 추정해 반듯하게 펴는 문제를 풀며,
대규모 합성/실측 문서 데이터로 학습되어 공개되어 있다. 이 모델들이 배운 것,
즉 "원근 왜곡된 평면 사각형의 경계와 자세를 읽는 표현"은 문서가 아닌 평면
패널에도 그대로 필요한 능력이다.

### 1.2 도메인 간극이라는 변수

전이가 항상 성공하는 것은 아니다. 원천 도메인(문서: 종이 질감, 텍스트, 접힘)과
표적 도메인(검사 패널: fringe 패턴, 반사, 균일 표면)의 외관 통계가 다르므로,
표현의 어느 수준까지 재사용 가능한지가 관건이다. 일반적으로 신경망의 앞쪽 층은
도메인에 무관한 범용 특징(엣지, 색 대비)을, 뒤쪽 층은 원천 도메인 특화 특징을
배우므로, 파인튜닝은 "앞은 보존하고 뒤를 조정하는" 방향으로 설계된다. 이 문서의
2.3절에서 그 구체적 기법들을 다룬다.

### 1.3 문제 정의

입력은 컬러 이미지 한 장이고, 출력은 TL(좌상단) $\to$ TR(우상단) $\to$ BR(우하단) $\to$
BL(좌하단) 시계 방향 순서의 코너 4개 좌표(정규화 [0, 1])이다. 출력 형식과
후처리는 직접 좌표 회귀와 동일하며, 이 방법론의 정체성은 "무엇을 예측하는가"가
아니라 "어떤 가중치에서 출발해 어떻게 적응하는가"에 있다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

**전이학습의 형식화.** 원천(source) 과제에서 학습된 파라미터를 $\theta_0$라
하자. 표적(target) 과제의 데이터 $\mathcal{D} = \{(I_n, Y_n)\}_{n=1}^{N}$은
작다($N$이 수십-수백). 학습은 무작위 초기값이 아니라 $\theta_0$에서 시작하는
최적화이다.

$$
\hat{\theta} = \arg\min_\theta \frac{1}{N} \sum_{n=1}^{N}
\mathcal{L}\big(f_\theta(I_n),\ Y_n\big), \qquad
\theta^{(0)} = \theta_0
$$

경사하강법은 초기값 근방의 해를 찾는 국소 탐색이므로, 좋은 $\theta_0$는 두 가지
효과를 낸다. 첫째, 원천 과제에서 배운 표현이 표적 과제에도 유효하다면 도달해야
할 해가 $\theta_0$에서 가깝고, 둘째, 적은 반복(적은 데이터)으로도 그 근방에
도달할 수 있다. 표적 데이터가 적을수록 이 초기값의 가치가 커진다.

모델 자체의 입출력은 직접 회귀와 같다.

$$
f_\theta: \mathbb{R}^{3 \times H \times W} \to [0, 1]^8, \qquad
\hat{\mathbf{y}} = (\hat{x}_1, \hat{y}_1, \dots, \hat{x}_4, \hat{y}_4)
$$

### 2.2 모델 구조

구조는 "사전학습 인코더 + 신규 회귀 head"의 결합이다.

**(1) 사전학습 인코더의 원형.** 문서 기하 모델의 대표 구조를 개관한다.

- **DocTr**: CNN이 이미지에서 특징 맵을 추출하고, transformer 인코더-디코더가
  특징 위치들 사이의 전역 관계를 모델링하여 문서의 왜곡장(각 픽셀이 펴진 문서의
  어디로 가야 하는지)을 예측한다. transformer의 핵심인 self-attention은 특징
  벡터 집합 $Z$를 질의/키/값 행렬 $Q = ZW_Q$, $K = ZW_K$, $V = ZW_V$로 사영한 뒤

$$
\mathrm{Attn}(Q, K, V) = \mathrm{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V
$$

  로 모든 위치 쌍의 유사도에 따라 정보를 섞는 연산이다. 문서 전체의 기하(경계
  네 변의 배치)는 멀리 떨어진 위치들의 관계이므로, 전역 상호작용을 직접 계산하는
  attention이 이 문제에 적합하다.
- **DocScanner**: 순환(recurrent) 구조로 왜곡장 추정을 반복 정제하는 모델로,
  거친 추정에서 시작해 잔차를 단계적으로 줄인다.

두 계열 모두 "왜곡된 평면 사각형의 기하를 읽는" 인코더 표현을 학습하며, 이
인코더가 전이의 대상이다.

**(2) 어댑터 구성.** 사전학습 모델의 최종 출력(왜곡장)은 본 과제의 출력(코너
8개)과 다르므로, 인코더까지만 재사용하고 그 위에 회귀 head를 새로 단다.

$$
f_\theta(I) = \sigma\Big( \mathrm{FC}\big(\mathrm{GAP}(\phi_{\theta_e}(I))\big) \Big)
$$

$\phi_{\theta_e}$는 사전학습 인코더(특징 맵 출력), GAP는 전역 평균 풀링(채널별
공간 평균), FC는 8차원 완전연결 계층, $\sigma$는 원소별 sigmoid이다. head의
파라미터는 무작위 초기화하고, 인코더 파라미터는 $\theta_0$에서 시작한다.

**전체 shape 흐름** (입력 $512 \times 512$, 배치 크기 $N$, 대표 차원):

| 단계 | 연산 | 출력 shape | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, 512, 512) | 컬러 이미지 배치 |
| CNN 스템 | 합성곱 계층 | (N, 256, 32, 32) | 국소 특징 |
| Transformer 인코더 | self-attention 블록 다수 | (N, 1024, 256) | 위치 간 전역 관계 반영 특징 (32x32=1024 토큰) |
| 재배열 + GAP | - | (N, 256) | 전역 요약 벡터 |
| FC (신규) | 선형 변환 | (N, 8) | 좌표 logits |
| Sigmoid | 원소별 $\sigma$ | (N, 8) | 정규화 좌표 예측 |

```text
이미지 (3x512x512)
   $\to$ [CNN 스템 (사전학습)]           $\to$ 특징 맵 (256x32x32)
   $\to$ [Transformer 인코더 (사전학습)] $\to$ 전역 관계 특징 (1024 토큰 x 256)
   $\to$ [GAP + FC (신규 head)]          $\to$ logits (8)
   $\to$ [sigmoid]                       $\to$ 정규화 좌표 (8)
```

### 2.3 손실 함수

회귀 손실 자체는 직접 회귀와 동일한 좌표 손실이다. 원소별 오차
$e = \hat{y}_i - y_i$에 SmoothL1($|e| < \beta$에서 $e^2/2\beta$, 그 외
$|e| - \beta/2$) 또는 소오차 그래디언트를 증폭하는 Wing Loss를 적용하고 8개
원소에 대해 평균한다.

$$
\mathcal{L}_{coord}(\hat{\mathbf{y}}, \mathbf{y})
= \frac{1}{8} \sum_{i=1}^{8} \ell(\hat{y}_i - y_i)
$$

이 방법론의 고유한 내용은 손실 자체가 아니라 **최적화의 제어**에 있다. 소량
데이터 파인튜닝에서는 "사전학습이 배운 것을 파괴하지 않으면서 필요한 만큼만
바꾸는" 장치가 성능을 좌우한다.

**(1) 단계적 동결 해제(freezing schedule).** 초기에는 인코더를 동결하고 head만
학습한다($\nabla_{\theta_e} = 0$으로 고정). head가 무작위 초기 상태일 때 발생하는
큰 그래디언트가 인코더 표현을 훼손하는 것을 막기 위함이다. head가 수렴한 뒤
인코더를 해제해 전체를 낮은 학습률로 미세 조정한다.

**(2) 층별 차등 학습률(discriminative fine-tuning).** 층 $l$ (총 $L$층, 출력에
가까울수록 큰 $l$)의 학습률을 기하급수로 차등한다.

$$
\eta_l = \eta \cdot \gamma^{\,L - l}, \qquad 0 < \gamma < 1
$$

앞쪽 층(범용 특징)은 거의 움직이지 않고 뒤쪽 층(도메인 특화 특징)은 크게
움직이도록 하는 설계로, "앞은 보존, 뒤는 조정"이라는 전이학습의 원칙을 학습률에
직접 반영한다.

**(3) 초기값 앵커 정규화(L2-SP).** 파라미터가 사전학습 값 $\theta_0$에서 멀어지는
것 자체에 벌점을 준다.

$$
\mathcal{L} = \mathcal{L}_{coord}
+ \frac{\mu}{2} \big\| \theta_e - \theta_e^0 \big\|_2^2
$$

통상의 가중치 감쇠(weight decay)가 파라미터를 0으로 끌어당기는 것과 달리, 이
항은 사전학습 해로 끌어당긴다. 소량 데이터가 만드는 과적합 방향의 이동을 억제해
파국적 망각(catastrophic forgetting, 파인튜닝 중 원천 과제에서 배운 능력이
소실되는 현상)을 완화한다.

**(4) few-shot 보조 수단.** 데이터가 수십 장 수준이면 강한 데이터 증강(기하 +
광학)과 이른 종료(early stopping, 검증 손실이 악화되기 시작하면 중단)를 병행해
실질 표본 다양성을 늘리고 과적합 시점을 넘기지 않도록 한다.

### 2.4 후처리 이론

raw 출력이 좌표 logits $(N, 8)$이므로 후처리는 직접 회귀와 동일한 결정적 변환
두 개이다. 원소별 sigmoid로 좌표를 열린 구간 $(0, 1)$에 가두고,

$$
\hat{\mathbf{y}} = \sigma(\mathbf{z}) \in (0, 1)^8
$$

8개 값을 4개 코너의 (x, y) 쌍으로 재배열한다.

$$
\hat{Y}_k = (\hat{y}_{2k-1},\ \hat{y}_{2k}), \qquad k = 1, \dots, 4
$$

학습 시 출력 원소와 코너의 대응이 손실로 고정되므로 출력 순서가 곧 코너 순서
이며, 정렬/매칭 단계와 실패 모드가 없다.

## 3. 결론

Document Pretrained는 모델 구조가 아니라 **지식의 출처**로 차별화되는 방법론
이다. 장점은 다음과 같다.

- **표본 효율**: 문서 기하를 이미 아는 표현에서 출발하므로, 소량의 표적 데이터로
  높은 출발점의 성능을 기대할 수 있다.
- **기하 구조의 일치**: 원천 과제(문서 펴기)와 표적 과제(패널 코너)가 "원근
  왜곡된 평면 사각형"이라는 동일한 기하 문제를 공유해, 전이 근거가 표면적 유사성
  보다 깊다.
- **후처리 자명성**: 직접 회귀와 동일한 출력 경로로, 실패 모드가 없다.

한계는 도메인 간극과 비용이다. 문서와 검사 패널의 외관 통계 차이가 크면 전이
이득이 줄고, 사전학습 모델(특히 transformer 계열)은 경량 CNN보다 무거워 CPU
배포에 불리하다. 또한 외부 패키지/가중치 의존이 재현성과 유지보수의 변수가 된다.

이 방법론은 실측 데이터가 극히 적은 조건에서의 대응책이자, "사전학습 표현이
어디까지 통하는가"를 재는 탐침으로서, 처음부터 학습하는 방법론들의 표본 효율과
비교하는 축을 제공한다.

## 부록 A. 데이터 인터페이스

- **preprocessor**: 표준 코너 $(N, 4, 2)$를 $(N, 8)$로 flatten한다. 좌표가 이미
  [0, 1] 정규화 및 TL/TR/BR/BL 순서 고정 상태이므로 모양 변경 외의 변환은 없다
  (direct와 동일).
- **postprocessor**: raw 출력 logits $(N, 8)$에 sigmoid를 적용한 뒤 $(N, 4, 2)$로
  reshape한다 (direct와 동일, 2.4절).
- **실패 모드**: 모델이 항상 8개 값을 출력하므로 실패 모드가 구조적으로 없다.
  별도의 fallback이 불필요하다.

## 부록 B. 학습 전략

3단계 학습 전략 중 1단계(공개 데이터 사전학습)를 공개 가중치 로드로 대체하는
것이 이 방법론의 정의적 특징이다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | DocTr/DocScanner 공개 가중치 로드 | 문서 기하 표현 확보 (학습 없음) | - |
| 1'. Head warmup | SmartDoc + MIDV-2020 (공개) | 인코더 동결, 신규 head만 학습 | 1e-3 (head) |
| 2. Domain Adaptation | 합성 fringe 패턴 | 인코더 해제, 층별 차등 학습률로 적응 | 1e-4 (base) |
| 3. Fine-tuning | 실측 데이터 (소량) | L2-SP + 강한 증강 + 이른 종료로 최종 적응 | 1e-5 (base) |

**층별 학습률**: 단계 2부터 $\eta_l = \eta \cdot \gamma^{L-l}$ ($\gamma \approx
0.8$)을 적용해 인코더 앞쪽 층의 이동을 억제한다 (2.3절).

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 8개 좌표 자유 회귀로 임의 볼록 사각형 표현 가능 (direct와 동일) |
| F2 (단일 객체, 50% 이상) | 사전학습 원천(문서 촬영) 자체가 단일 대형 사각형 조건이라 전이 분포가 일치 |
| F3 (코너 경계 내부) | sigmoid 치역 $(0, 1)$로 구조적 보장 (2.4절) |
| F4 (실측 소량) | 이 방법론의 존재 이유. 사전학습 가중치 + few-shot 파인튜닝 기법(2.3절)으로 정면 대응 |
| F5 (서브픽셀 정밀도) | 전역 회귀 경로라 direct와 같은 한계. Wing Loss 및 사전학습 표현의 품질로 부분 완화 |
| F6 (CPU 배포) | transformer 계열 인코더로 무거운 편. F6 관점에서는 하위 후보이며, 성능 상한 확인 후 경량화 판단 |
| F7 (조명/글레어 변동) | 광학 증강 적용. 문서 사전학습이 다양한 촬영 조명을 이미 포함해 부분적 내성 기대 |
| F8 (가림 없음) | 문서 도메인도 가림 없는 단일 객체가 일반적이라 전제 일치. 특별한 대응 불필요 |
