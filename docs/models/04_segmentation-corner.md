# Segmentation Corner (seg)

## 초록

Segmentation Corner는 코너 좌표를 신경망이 직접 출력하는 대신, 이미지의 각 픽셀이
사각형 내부인지 외부인지를 분류하는 세그멘테이션(segmentation) 문제로 우회한 뒤,
예측된 마스크의 윤곽선에서 기하 연산으로 4개 코너를 추출하는 방법론이다. 학습 신호가
이미지당 8개 숫자가 아니라 픽셀 수만큼(수십만 개) 주어지므로 감독(supervision)이
조밀하고, 완전 합성곱(fully convolutional) 구조가 입력과 출력의 공간 대응을 끝까지
유지하므로 경계 형상이 출력에 그대로 반영된다. 코너는 학습이 끝난 뒤 이진화된
마스크에 컨투어 추출(findContours)과 폴리곤 근사(approxPolyDP)를 적용해 얻는다.
경계 대비가 선명한 도메인에서 안정적으로 동작하는 반면, 후처리가 4개의 꼭짓점을
항상 보장하지는 않으므로 실패 모드 처리가 필요하다는 점이 직접 회귀 계열과의 가장
큰 차이이다.

## 1. 서론

### 1.1 배경: 좌표를 직접 맞히지 않고 영역을 칠하는 이유

코너 검출을 회귀로 정식화하면 모델은 이미지 한 장에서 정답 숫자 8개를 배운다. 학습
표본 하나가 제공하는 오차 신호가 8개뿐이라는 뜻이다. 반면 같은 정답 코너로부터
"사각형 내부 픽셀은 1, 외부 픽셀은 0"인 마스크를 만들면, 이미지 한 장이 픽셀 수만큼
(예: $512 \times 512 = 262{,}144$개)의 정답 신호를 제공한다. 이렇게 조밀한 감독은
특히 학습 데이터가 적을 때 모델이 물체의 형상을 안정적으로 배우도록 돕는다.

또 하나의 근거는 공간 정보의 보존이다. 직접 회귀 모델은 전역 평균 풀링(GAP)으로
공간 격자를 요약하는 순간 "어디에" 정보가 간접 부호화된다. 세그멘테이션 모델은
출력 자체가 입력과 같은 격자 위의 지도(map)이므로, 사각형 경계의 위치가 출력에
픽셀 단위로 직접 나타난다. 코너는 그 경계 정보로부터 기하 연산으로 복원한다.

### 1.2 세그멘테이션은 픽셀 단위 분류다

분류(classification)는 입력 하나에 범주 하나를 할당하는 문제이다. 세그멘테이션은
이 분류를 모든 픽셀에 대해 동시에 수행하는 문제로, "이 픽셀은 사각형 내부인가
아닌가"라는 이진 분류를 $H \times W$번 푸는 것과 같다. 따라서 출력은 각 픽셀이
내부일 확률을 담은 확률 지도(probability map)이고, 분류에서 쓰는 손실 함수(교차
엔트로피)를 픽셀마다 적용해 학습한다.

### 1.3 문제 정의

입력은 컬러 이미지 한 장이다. 중간 출력은 사각형 영역의 이진 마스크(정확히는 확률
지도)이고, 최종 출력은 TL(좌상단) $\to$ TR(우상단) $\to$ BR(우하단) $\to$ BL(좌하단) 시계
방향 순서의 코너 4개 좌표이다. 각 좌표는 이미지 크기로 나눈 [0, 1] 정규화 좌표로
표현한다. 전체 파이프라인은 "학습되는 세그멘테이션"과 "학습되지 않는 기하 후처리"의
2단계 직렬 구성이다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

입력 이미지를 $I \in \mathbb{R}^{3 \times H \times W}$, 정답 코너를
$Y \in [0, 1]^{4 \times 2}$로 표기한다. 코너 4개가 이루는 볼록 사각형 영역을
$Q(Y)$라 하면, 정답 마스크 $M \in \{0, 1\}^{H \times W}$는 각 픽셀 중심이 영역에
포함되는지로 정의된다 (이 변환을 래스터화라 한다).

$$
M_{ij} =
\begin{cases}
1 & \text{픽셀 } (i, j) \text{의 중심이 } Q(Y) \text{ 내부} \\
0 & \text{그 외}
\end{cases}
$$

학습으로 얻고자 하는 것은 마스크의 확률 지도를 출력하는 함수이다.

$$
f_\theta: \mathbb{R}^{3 \times H \times W} \to [0, 1]^{H \times W}
$$

$f_\theta(I)$의 각 원소 $P_{ij}$는 "픽셀 $(i, j)$가 사각형 내부일 확률"이다. 최종
코너는 학습 파라미터가 없는 후처리 함수 $g$를 통해 얻는다.

$$
\hat{Y} = g\big(\mathbb{1}[P \ge \tau]\big) \in [0, 1]^{4 \times 2}
$$

여기서 $\mathbb{1}[\cdot]$는 조건이 참이면 1, 거짓이면 0인 지시 함수이고 $\tau$는
이진화 임계값(보통 0.5)이다. $g$의 내용(컨투어 추출과 폴리곤 근사)은 2.4절에서
정의한다. 정리하면 이 방법론은 "코너 $\to$ 마스크"로 문제를 바꿔 학습하고, "마스크 $\to$
코너"로 되돌리는 역변환을 후처리에 맡기는 구조이다.

### 2.2 모델 구조

픽셀마다 예측을 내려면 출력이 입력에 대응하는 공간 해상도의 지도여야 한다. 이를 위해
인코더-디코더(encoder-decoder) 구조, 그중에서도 U-Net 계열을 사용한다. 인코더가
해상도를 줄이며 의미(semantic) 정보를 모으고, 디코더가 해상도를 되살리며 위치
정보를 복원한다.

**(1) 인코더와 다운샘플링.** 인코더는 합성곱 계층과 풀링(pooling)의 반복이다.
합성곱은 작은 필터를 이미지 위에서 이동시키며 국소 가중합을 계산하는 연산이고,
최대 풀링(max pooling)은 $s \times s$ 창 안에서 최댓값 하나만 남겨 해상도를
$1/s$로 줄이는 연산이다.

$$
\mathrm{MaxPool}(X)_{c,i,j} = \max_{0 \le a, b < s} X_{c,\ si + a,\ sj + b}
$$

해상도를 줄일 때마다 한 픽셀이 바라보는 입력 영역(수용 영역, receptive field)이
넓어지므로, 깊은 층일수록 "이 근방이 사각형 내부인가"라는 넓은 문맥 판단이
가능해진다. 대신 위치 정보는 점점 거칠어진다.

**(2) 디코더와 업샘플링.** 디코더는 줄어든 특징 맵의 해상도를 다시 키운다. 대표적
방법은 두 가지이다. 쌍선형 보간(bilinear interpolation)은 이웃 4개 픽셀 값의
가중 평균으로 중간 값을 채우는 고정 연산이고, 전치 합성곱(transposed convolution)은
업샘플링 자체를 학습 가능한 필터로 수행하는 연산이다. 전치 합성곱은 입력의 각
값을 필터 모양으로 "찍어서" 넓은 출력에 누적하는 것으로 이해할 수 있으며, stride
$s$일 때 해상도가 $s$배가 된다.

**(3) skip connection.** 인코더의 깊은 층은 문맥은 풍부하지만 위치가 거칠고, 얕은
층은 위치는 정확하지만 문맥이 얕다. U-Net은 디코더의 각 단계에 같은 해상도의
인코더 특징 맵을 채널 방향으로 이어 붙여(concatenate) 두 정보를 결합한다.

$$
D_l = \mathrm{Conv}\big(\big[\mathrm{Up}(D_{l+1});\ E_l\big]\big)
$$

여기서 $E_l$은 인코더 $l$단계 특징 맵, $D_l$은 디코더 $l$단계 출력, $[\cdot;\cdot]$는
채널 연결이다. 이 연결 덕분에 풀링으로 잃었던 경계의 미세 위치가 디코더에서
복원되어, 마스크 경계가 선명해진다. 코너 추출의 정밀도는 경계 품질에 직결되므로
이 구조가 본 방법론의 핵심이다.

**(4) 출력 head와 sigmoid.** 디코더 최종 특징 맵에 $1 \times 1$ 합성곱을 적용해
채널을 1로 줄이면 픽셀별 logit 지도 $Z \in \mathbb{R}^{H \times W}$가 된다. 각
logit에 sigmoid 함수 $\sigma(z) = 1 / (1 + e^{-z})$를 원소별로 적용하면 확률 지도
$P = \sigma(Z)$를 얻는다.

**전체 shape 흐름** (입력 $512 \times 512$, 배치 크기 $N$, ResNet50 인코더 기준):

| 단계 | 연산 | 출력 shape | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, 512, 512) | 컬러 이미지 배치 |
| 인코더 0 | conv1 + BN + ReLU | (N, 64, 256, 256) | 저수준 패턴 (선, 모서리), 1/2 |
| 인코더 1 | maxpool + layer1 | (N, 256, 128, 128) | 저수준 형상, 1/4 |
| 인코더 2 | layer2 | (N, 512, 64, 64) | 중간 수준 패턴, 1/8 |
| 인코더 3 | layer3 | (N, 1024, 32, 32) | 부분 형상, 1/16 |
| 인코더 4 | layer4 | (N, 2048, 16, 16) | 전역 문맥 (사각형 여부), 1/32 |
| 디코더 1 | Up + skip(인코더 3) + Conv | (N, 256, 32, 32) | 문맥 + 위치 결합, 1/16 |
| 디코더 2 | Up + skip(인코더 2) + Conv | (N, 128, 64, 64) | 경계 복원, 1/8 |
| 디코더 3 | Up + skip(인코더 1) + Conv | (N, 64, 128, 128) | 미세 경계 복원, 1/4 |
| 디코더 4 | Up + skip(인코더 0) + Conv | (N, 32, 256, 256) | 미세 경계 복원, 1/2 |
| 출력 | 1x1 Conv | (N, 1, 256, 256) | 픽셀별 logit 지도, 1/2 |

```text
이미지 (3x512x512)
   $\to$ [ResNet 인코더: Conv+Pool 반복] $\to$ 문맥 특징 (2048x16x16)
   $\to$ [UNet 디코더: Up+skip 반복]     $\to$ 경계 복원 특징 (32x256x256)
   $\to$ [1x1 Conv]                      $\to$ 마스크 logits (1x256x256)
   $\to$ [sigmoid]                       $\to$ 확률 지도 P (1x256x256)
```

인코더는 ImageNet 사전학습 ResNet(기본 resnet50)이며 stage 5개(conv1, layer1~layer4)
에서 skip 특징을 뽑는다. 디코더는 전치 합성곱으로 해상도를 4단 되살리며 각 단계에서
같은 해상도의 인코더 특징을 이어 붙인다. 출력은 입력의 1/2 해상도(입력 512이면
256, 224이면 112)이다. 전체 해상도가 아니라 절반으로 두는 것은 마스크 타깃 생성과
디코더 계산의 메모리/비용을 줄이기 위한 선택이며, 마스크 해상도가 입력 크기에
비례하므로 image_size를 키우면 경계 분해능도 함께 올라간다(2.3절의 정밀도 상한 참조).

**이 구조가 의미하는 것**: 직접 회귀가 GAP로 공간 격자를 버리는 지점에서, 이
구조는 반대로 공간 격자를 출력까지 운반한다. 그 대가로 출력 텐서가 커서 메모리와
계산 비용이 늘고, 좌표를 얻으려면 별도의 후처리 단계가 필요해진다.

### 2.3 손실 함수

**픽셀별 이진 교차 엔트로피(BCE, Binary Cross-Entropy).** 픽셀 하나의 예측 확률
$P_{ij}$와 정답 $M_{ij} \in \{0, 1\}$에 대한 교차 엔트로피는 다음과 같다.

$$
\mathcal{L}_{\mathrm{BCE}} = -\frac{1}{HW} \sum_{i=1}^{H} \sum_{j=1}^{W}
\Big[ M_{ij} \log P_{ij} + (1 - M_{ij}) \log (1 - P_{ij}) \Big]
$$

정답이 1인 픽셀에서는 $-\log P_{ij}$가 되어 확률을 1로 밀고, 정답이 0인 픽셀에서는
$-\log(1 - P_{ij})$가 되어 확률을 0으로 민다. sigmoid와 BCE를 결합하면 logit
$z_{ij}$에 대한 그래디언트가 다음처럼 단순해진다.

$$
\frac{\partial \mathcal{L}_{\mathrm{BCE}}^{(ij)}}{\partial z_{ij}} = P_{ij} - M_{ij}
$$

즉 "예측 확률과 정답의 차이"가 그대로 교정 신호가 되는 매끄러운 손실이다. 실제
구현에서는 수치 안정성을 위해 sigmoid와 BCE를 합쳐 계산하는 형태(BCE with logits)를
사용한다.

**Dice 손실.** BCE는 픽셀을 독립적으로 취급하므로, 영역 전체가 정답과 얼마나
겹치는지는 직접 재지 않는다. Dice 계수는 두 영역의 겹침을 하나의 수치로 잰다.

$$
\mathrm{Dice}(P, M) =
\frac{2 \sum_{i,j} P_{ij} M_{ij} + \epsilon}
{\sum_{i,j} P_{ij} + \sum_{i,j} M_{ij} + \epsilon},
\qquad
\mathcal{L}_{\mathrm{Dice}} = 1 - \mathrm{Dice}(P, M)
$$

분자는 예측과 정답의 교집합(의 2배), 분모는 두 영역 크기의 합이며, $\epsilon$은
분모가 0이 되는 것을 막는 작은 상수이다. 완전히 겹치면 1, 전혀 겹치지 않으면 0이
된다. Dice는 영역 크기에 대해 상대적인 척도이므로, 전경과 배경의 픽셀 수가 크게
불균형해도 전경 겹침을 공정하게 평가한다. 본 문제는 사각형이 화면의 50% 이상을
차지하는 조건이라 불균형이 심하지 않지만, Dice를 함께 쓰면 "영역 수준의 일관성"
이라는 BCE가 놓치는 관점이 보강된다.

**결합 손실.** 두 손실을 가중합한다.

$$
\mathcal{L} = \mathcal{L}_{\mathrm{BCE}} + \lambda \, \mathcal{L}_{\mathrm{Dice}}
$$

$\lambda$는 보통 1로 두어 두 항의 균형을 맞춘다. BCE가 픽셀 단위의 안정적
그래디언트를 제공하고, Dice가 영역 단위의 겹침을 끌어올리는 상호 보완 구성이다.

**경계와 손실의 관계.** 코너 추출 관점에서 중요한 픽셀은 경계 근방의 소수이다.
내부와 외부 깊숙한 픽셀은 어떤 합리적 모델이든 쉽게 맞히므로, 학습 후반의 손실은
사실상 경계 픽셀의 오차가 지배한다. 출력 해상도를 높게 유지할수록 경계 픽셀의
비중과 위치 분해능이 좋아지므로, 세그멘테이션 기반 코너 검출에서는 출력 해상도가
정밀도의 상한을 정한다.

### 2.4 후처리 이론

후처리 $g$는 이진 마스크에서 코너 4개를 복원하는 결정적(비학습) 절차이며, 다음
다섯 단계로 구성된다.

**1단계: 이진화.** 확률 지도를 임계값 $\tau = 0.5$로 자른다.

$$
B_{ij} = \mathbb{1}[P_{ij} \ge \tau]
$$

**2단계: 컨투어 추출 (findContours).** 이진 이미지에서 전경 영역의 외곽 경계를
따라가며 순서 있는 점열을 추출한다. 표준 알고리즘은 Suzuki-Abe의 경계 추적(border
following)으로, 래스터 격자를 훑다가 전경-배경 전환점을 만나면 그 경계를 시계
방향으로 한 바퀴 따라가 폐곡선 $C = (p_1, p_2, \dots, p_n)$을 얻는다. 마스크에
잡음 조각이 여러 개 있을 수 있으므로, 면적이 최대인 컨투어 하나를 선택한다
(단일 객체 전제, 부록 C의 F2/F8 참조).

**3단계: 폴리곤 근사 (approxPolyDP).** 컨투어는 수백 개의 점으로 이루어진
곡선이므로, 이를 꼭짓점 몇 개의 다각형으로 단순화해야 한다. Douglas-Peucker
알고리즘은 다음 재귀 절차로 점을 솎아 낸다.

1. 곡선의 양 끝점을 잇는 현(직선)을 그린다.
2. 곡선 위의 점 중 현에서 가장 먼 점 $p^*$와 그 수직 거리 $d^*$를 찾는다. 점
   $p = (x_0, y_0)$과 직선 $ax + by + c = 0$ 사이의 거리는 다음과 같다.

$$
d(p, l) = \frac{|a x_0 + b y_0 + c|}{\sqrt{a^2 + b^2}}
$$

3. $d^* > \varepsilon$이면 $p^*$를 꼭짓점으로 채택하고, 곡선을 $p^*$에서 둘로
   나누어 각각에 같은 절차를 재귀 적용한다. $d^* \le \varepsilon$이면 이 구간의
   중간 점을 모두 버린다.

허용 오차 $\varepsilon$이 클수록 꼭짓점이 적어진다. 사각형 마스크라면 이상적으로
꼭짓점 4개가 남지만, $\varepsilon$이 너무 작으면 경계 잡음이 살아남아 5개 이상,
너무 크면 실제 꼭짓점이 병합되어 3개 이하가 될 수 있다. 그래서 $\varepsilon$을
컨투어 둘레에 비례하게 두고($\varepsilon = \alpha \cdot \mathrm{Perimeter}(C)$,
$\alpha \approx 0.02$), 결과가 4점이 아니면 $\alpha$를 키우거나 줄이며 수 회
재시도한다.

**4단계: 코너 순서 정렬.** 근사로 얻은 4점의 순서는 컨투어 시작점에 따라
달라지므로 표준 순서(TL $\to$ TR $\to$ BR $\to$ BL)로 정렬한다. 무게중심
$c = \frac{1}{4} \sum_k p_k$를 기준으로 각 점의 각도
$\phi_k = \mathrm{atan2}(y_k - c_y,\ x_k - c_x)$를 계산해 시계 방향으로 돌린 뒤,
$x + y$가 가장 작은 점(좌상단에 가장 가까운 점)이 첫 번째가 되도록 순환 이동한다.

**5단계: 정규화와 유효성 검증.** 픽셀 좌표를 이미지 크기로 나누어 [0, 1] 좌표로
바꾸고, 퇴화 여부(코너 간 최소 거리가 임계값 미만인지)를 검사한다.

**실패 모드.** 이 후처리는 다음 세 경우에 4점 복원에 실패할 수 있다.
(i) 마스크가 비어 있거나 유효한 컨투어가 없는 경우, (ii) $\varepsilon$ 재시도
후에도 꼭짓점이 4개로 수렴하지 않는 경우, (iii) 4점이 나왔지만 퇴화 판정에 걸리는
경우이다. 직접 회귀가 구조적으로 항상 8개 값을 출력하는 것과 달리, 세그멘테이션
경로는 "검출 실패"라는 상태가 존재하므로 성공 플래그를 함께 반환해야 한다.

**정밀도의 상한.** 컨투어 점은 픽셀 격자 위에 있으므로 코너 좌표의 분해능은
마스크 해상도로 제한된다. 서브픽셀 정밀도가 필요하면 추출된 코너를 초기값으로
그래디언트 기반 정제(cornerSubPix)를 후단에 붙이는 선택지가 있다.

## 3. 결론

Segmentation Corner는 코너 검출을 "영역 칠하기 + 기하 복원"의 2단계로 분해한
방법론이다. 장점은 다음과 같다.

- **조밀한 감독**: 이미지당 픽셀 수만큼의 학습 신호가 있어, 적은 데이터에서도
  형상 학습이 안정적이다.
- **공간 대응 보존**: 완전 합성곱 구조가 경계 위치를 출력 격자에 직접 표현하므로,
  경계 대비가 선명한 도메인에서 강하다.
- **해석 용이성**: 중간 산출물(마스크)을 눈으로 확인할 수 있어, 실패 원인이
  세그멘테이션과 후처리 중 어디에 있는지 진단하기 쉽다.

한계는 후처리의 취약성과 해상도 제한이다. 폴리곤 근사는 4점을 보장하지 않으므로
실패 모드 처리와 성공률(SR) 추적이 필수이고, 코너 분해능이 마스크 해상도에 묶이므로
서브픽셀 정밀도를 위해서는 별도 정제가 필요하다. 또한 출력이 조밀한 만큼 직접
회귀보다 메모리와 계산 비용이 크다.

이 방법론은 물체 경계와 배경의 대비가 뚜렷하고 단일 물체가 화면 대부분을 차지하는
조건에서 가장 적합하며, 직접 회귀 베이스라인과 상보적인 비교 축(조밀 감독 대 희소
감독)을 제공한다.

## 부록 A. 데이터 인터페이스

- **preprocessor**: 표준 코너 $(N, 4, 2)$를 채운 볼록 사각형 마스크 $(N, 1, H, W)$로
  래스터화한다 (2.1절의 $M$). 정규화 좌표를 마스크 픽셀 좌표로 되돌린 뒤, 네 변에
  대한 반평면(half-plane) 내부 판정을 순수 텐서 연산으로 배치 단위로 수행해 GPU에서
  마스크를 채운다(외부 라이브러리 없이 torch 연산으로 구현). 마스크 크기 $H$는 모델
  출력 해상도(입력의 1/2)에서 받아 매 스텝 생성하므로, image_size가 바뀌어도 타깃이
  출력과 정합한다.
- **postprocessor**: raw 출력 logits $(N, 1, H, W)$에 sigmoid와 임계값 0.5 이진화를
  적용하고, findContours(최대 면적 컨투어 선택) + approxPolyDP($\varepsilon$ 반복
  조정) + `order_corners` 정렬 + 정규화를 거쳐 $(N, 4, 2)$를 만든다 (2.4절).
  4점 근사 실패 시 minAreaRect, `mask_to_corners`(극점 기반) 순으로 폴백한다.
- **실패 모드**: 컨투어 부재, 4점 수렴 실패, 퇴화 판정(`is_invalid_corners`)의 세
  경우 해당 표본의 코너를 NaN으로 반환한다. Evaluator의 `SuccessRate` 메트릭이 NaN이
  아닌 표본 비율로 SR(Success Rate)을 집계하며, 정확도 메트릭은 성공 표본에 대해서만
  계산하고 항상 SR과 함께 보고한다 (README.md 4절).

## 부록 B. 학습 전략

3단계 학습 전략을 적용한다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | SmartDoc + MIDV-2020 (공개) | 일반적인 사각형 영역 분할 능력 학습 | 1e-3 |
| 2. Domain Adaptation | 합성 fringe 패턴 | 검사 도메인의 화면 특성에 적응 | 1e-4 |
| 3. Fine-tuning | 실측 데이터 (소량) | 최종 경계 정밀도 확보 | 1e-5 |

**타깃 생성이 자동이라는 이점**: 마스크 타깃은 preprocessor가 코너 CSV로부터 매번
생성하므로, 어떤 단계의 데이터든 코너 라벨만 있으면 별도의 마스크 어노테이션 없이
사용할 수 있다.

**인코더 초기화**: 인코더는 ImageNet 사전학습 가중치에서 시작한다. 조밀한 감독
덕분에 무작위 초기화로도 학습이 가능하지만, 사전학습 초기화가 수렴 속도와 소량
데이터 일반화에 유리하다.

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 마스크는 임의 형상을 픽셀 단위로 표현하므로 볼록 사각형을 근사 없이 담는다 |
| F2 (단일 객체, 50% 이상) | 최대 면적 컨투어 선택이 안정적으로 동작하는 근거. 배경의 소규모 오검출 조각이 자동 배제됨 |
| F3 (코너 경계 내부) | 사각형이 이미지 테두리에 걸리지 않으므로 컨투어가 잘리지 않고 온전한 폐곡선으로 추출됨 |
| F4 (실측 소량) | 3단계 학습 전략(부록 B)에 더해, 픽셀 단위 조밀 감독이 소량 데이터 학습을 보강 |
| F5 (서브픽셀 정밀도) | 컨투어가 픽셀 격자에 묶여 분해능 한계 존재 (2.4절). cornerSubPix 후정제 옵션으로 완화 |
| F6 (CPU 배포) | 인코더-디코더 구조로 직접 회귀보다 무거움. 입력/출력 해상도 축소로 비용 조절, 후처리 자체는 CPU 경량 |
| F7 (조명/글레어 변동) | 광학 증강(ColorJitter, GaussianBlur, GaussianNoise) 적용. 마스크 타깃은 광학 변화에 불변이므로 증강과 궁합이 좋음 |
| F8 (가림 없음) | 전경이 단일 연결 영역이라는 가정, 즉 최대 컨투어 하나만 처리하는 단순화의 근거 |
