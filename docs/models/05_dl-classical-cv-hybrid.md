# DL + Classical CV Hybrid (hybrid)

## 초록

DL + Classical CV Hybrid는 "대략 어디인가"는 딥러닝이, "정확히 어디인가"는 고전
컴퓨터 비전이 담당하는 2단계 방법론이다. 1단계에서 경량 세그멘테이션 신경망
(MobileNetV3 인코더 + U-Net 디코더)이 사각형 영역의 마스크를 예측하고, 2단계에서
학습이 필요 없는 고전 CV 연쇄(Canny 엣지 검출 $\to$ Hough 직선 변환 $\to$ 직선 교점 $\to$
cornerSubPix 서브픽셀 정제)가 마스크 경계에서 코너를 정밀하게 계산한다. 딥러닝의
강건한 의미적 위치 파악과 고전 CV의 결정적이고 값싼 기하 정밀화를 각자 잘하는
자리에 배치한 설계로, 학습 파라미터가 세그멘테이션 단계에만 있어 모델이 작고 CPU
추론에 유리하며, cornerSubPix가 서브픽셀 정밀도를 제공한다. 대가는 후처리 단계가
많아 실패 지점이 다양하다는 것이다.

## 1. 서론

### 1.1 배경: 두 패러다임의 상보성

딥러닝과 고전 CV는 강점의 방향이 다르다.

- **딥러닝**: 조명 변화, 반사, 복잡한 배경 속에서도 "무엇이 어디쯤 있는가"를
  데이터로부터 배워 강건하게 찾는다. 대신 출력 정밀도는 표현(격자 해상도 등)에
  묶이고, 동작이 통계적이라 예측 근거를 보장하기 어렵다.
- **고전 CV**: 그래디언트, 직선, 교점 같은 기하 연산은 결정적이고 수치적으로
  정밀하며 계산이 싸다. 대신 입력이 지저분하면(잡음, 저대비, 복잡한 배경) 쉽게
  깨진다.

하이브리드 설계는 이 상보성을 그대로 파이프라인에 옮긴다. 딥러닝이 지저분한
원본 이미지를 "사각형 내부 = 1"의 깨끗한 마스크로 정제해 주면, 고전 CV는 이상적
입력을 받아 자신 있는 기하 연산만 수행한다. 고전 CV가 깨지는 조건을 딥러닝이
제거해 주는 구조이다.

### 1.2 문제 정의

입력은 컬러 이미지 한 장이고, 최종 출력은 TL(좌상단) $\to$ TR(우상단) $\to$ BR(우하단)
$\to$ BL(좌하단) 시계 방향 순서의 코너 4개 좌표(정규화 [0, 1])이다. 중간 산출물은
사각형 영역의 마스크이며, 학습은 마스크 예측 단계에만 적용된다. 코너는 마스크로
부터 고전 CV 연쇄로 계산한다.

## 2. 이론 및 아키텍처

### 2.1 문제 정식화

입력 이미지를 $I \in \mathbb{R}^{3 \times H \times W}$, 정답 코너를
$Y \in [0, 1]^{4 \times 2}$로 표기한다. 정답 코너가 이루는 사각형 영역의 이진
마스크를 $M \in \{0, 1\}^{H \times W}$라 한다 (픽셀 중심이 사각형 내부이면 1).

학습되는 부분은 마스크 확률 지도를 출력하는 함수 하나이다.

$$
f_\theta: \mathbb{R}^{3 \times H \times W} \to [0, 1]^{H \times W}
$$

학습되지 않는 후처리는 네 개의 결정적 함수의 합성이다.

$$
g = g_{\mathrm{subpix}} \circ g_{\mathrm{intersect}} \circ g_{\mathrm{hough}}
\circ g_{\mathrm{canny}}
$$

전체 예측기는 $\hat{Y} = g\big(\mathbb{1}[f_\theta(I) \ge \tau]\big)$이다
($\mathbb{1}[\cdot]$는 지시 함수, $\tau$는 이진화 임계값). 각 $g$의 수학적
정의는 2.4절에서 다룬다. 이 정식화의 특징은 학습 문제(마스크 회귀)와 기하 문제
(코너 계산)가 완전히 분리되어, 각각을 독립적으로 진단하고 교체할 수 있다는
점이다.

### 2.2 모델 구조

학습되는 1단계는 경량성이 설계 목표이므로, 인코더로 MobileNetV3, 디코더로 U-Net
스타일 업샘플링 경로를 쓴다.

**(1) 깊이별 분리 합성곱(depthwise separable convolution).** MobileNet 계열
경량화의 핵심이다. 표준 합성곱은 공간 방향과 채널 방향을 한 번에 섞으며, 커널
$k \times k$, 입력 채널 $C_{in}$, 출력 채널 $C_{out}$, 특징 맵 $H \times W$일 때
곱셈 횟수가 다음과 같다.

$$
\mathrm{Cost}_{std} = k^2 \, C_{in} \, C_{out} \, H W
$$

깊이별 분리 합성곱은 이를 두 단계로 나눈다. 먼저 채널마다 독립적인 $k \times k$
필터를 적용하고(depthwise), 그다음 $1 \times 1$ 합성곱으로 채널을 섞는다
(pointwise).

$$
\mathrm{Cost}_{sep} = k^2 \, C_{in} \, H W + C_{in} \, C_{out} \, H W
$$

비용 비율은

$$
\frac{\mathrm{Cost}_{sep}}{\mathrm{Cost}_{std}}
= \frac{1}{C_{out}} + \frac{1}{k^2}
$$

로, $3 \times 3$ 커널에서 약 8-9배의 절감이다. 이 절감이 CPU 추론 예산의
근거이다.

**(2) MobileNetV3의 보조 구성.** 채널별 중요도를 학습해 곱하는
squeeze-and-excitation(전역 평균으로 채널 요약 $\to$ 작은 FC로 채널별 가중치 산출
$\to$ 원 특징에 곱함)과, 계산이 싼 비선형 활성화 hard-swish

$$
\mathrm{hswish}(x) = x \cdot \frac{\mathrm{ReLU6}(x + 3)}{6}, \qquad
\mathrm{ReLU6}(x) = \min(\max(0, x), 6)
$$

를 사용해 적은 비용으로 표현력을 보강한다.

**(3) U-Net 디코더.** 인코더의 중간 특징 맵들을 skip connection으로 받아,
업샘플링(쌍선형 보간 + 합성곱)을 반복하며 해상도를 복원한다. 인코더의 깊은 층은
문맥이 풍부하지만 위치가 거칠고 얕은 층은 그 반대이므로, 같은 해상도의 인코더
특징을 디코더에 이어 붙이는 skip connection이 경계의 미세 위치를 복원한다.
디코더 채널 수를 작게 유지해 경량성을 지킨다. 최종 $1 \times 1$ 합성곱이 1채널
마스크 logits를 만들고, 원소별 sigmoid $\sigma(z) = 1/(1+e^{-z})$로 확률 지도를
얻는다.

**전체 shape 흐름** (입력 $512 \times 512$, 배치 크기 $N$, 대표 채널 구성):

| 단계 | 연산 | 출력 shape | 의미 |
|---|---|---|---|
| 입력 | - | (N, 3, 512, 512) | 컬러 이미지 배치 |
| 인코더 1-2 | 분리 합성곱 + stride | (N, 24, 128, 128) | 저수준 패턴 |
| 인코더 3-4 | 분리 합성곱 + SE | (N, 48, 64, 64) | 중간 수준 패턴 |
| 인코더 5 | 분리 합성곱 + SE | (N, 96, 32, 32) | 전역 문맥 |
| 디코더 | Up + skip + Conv 반복 | (N, 16, 256, 256) | 경계 복원 |
| 출력 | Up + 1x1 Conv | (N, 1, 512, 512) | 마스크 logits |

```text
이미지 (3x512x512)
   $\to$ [MobileNetV3 인코더]         $\to$ 문맥 특징 (96x32x32)
   $\to$ [U-Net 디코더 (skip 결합)]   $\to$ 마스크 logits (1x512x512)
   $\to$ [sigmoid + 이진화]           $\to$ 이진 마스크 B
   $\to$ [Canny $\to$ Hough $\to$ 교점 $\to$ cornerSubPix]  $\to$ 코너 (4x2)
```

### 2.3 손실 함수

학습 대상이 세그멘테이션 단계뿐이므로, 손실은 마스크 예측에만 정의된다. 픽셀별
이진 교차 엔트로피(BCE)와 영역 겹침 기반 Dice 손실의 가중합을 쓴다. $P = f_\theta(I)$,
$M$을 각각 예측 확률 지도와 정답 마스크라 하면

$$
\mathcal{L}_{\mathrm{BCE}} = -\frac{1}{HW} \sum_{i,j}
\Big[ M_{ij} \log P_{ij} + (1 - M_{ij}) \log (1 - P_{ij}) \Big]
$$

$$
\mathcal{L}_{\mathrm{Dice}} = 1 -
\frac{2 \sum_{i,j} P_{ij} M_{ij} + \epsilon}
{\sum_{i,j} P_{ij} + \sum_{i,j} M_{ij} + \epsilon}
$$

$$
\mathcal{L} = \mathcal{L}_{\mathrm{BCE}} + \lambda \, \mathcal{L}_{\mathrm{Dice}},
\qquad \lambda = 1
$$

BCE는 픽셀 단위의 매끄러운 그래디언트(logit에 대해 $P_{ij} - M_{ij}$)를 제공하고,
Dice는 영역 수준의 겹침을 직접 최적화한다. 후단의 고전 CV가 소비하는 것은 마스크의
**경계 품질**이므로, 내부/외부 깊숙한 픽셀보다 경계 근방 픽셀의 정확도가 실질
성능을 좌우한다. 경계가 흐릿하면 Canny/Hough 단계에서 직선이 여러 개로 갈라지거나
끊기므로, 출력 해상도를 유지하고 경계가 선명해지도록 학습하는 것이 이 방법론의
손실 설계 목표이다.

### 2.4 후처리 이론

후처리는 이진 마스크 $B$에서 코너를 계산하는 고전 CV 연쇄이다. 각 단계를
수학적으로 정의한다.

**(a) Canny 엣지 검출.** 마스크(또는 확률 지도)의 경계 픽셀을 얇은 곡선으로
추출한다. 네 단계로 구성된다.

1. **가우시안 평활화**: 잡음 억제를 위해 가우시안 커널
   $G_\sigma(x, y) = \frac{1}{2\pi\sigma^2} \exp\big(-\frac{x^2 + y^2}{2\sigma^2}\big)$
   와 합성곱한다.
2. **그래디언트 계산**: Sobel 필터로 수평/수직 미분 $G_x, G_y$를 구하고, 크기와
   방향을 계산한다.

$$
\|\nabla I\| = \sqrt{G_x^2 + G_y^2}, \qquad
\phi = \mathrm{atan2}(G_y, G_x)
$$

3. **비최대 억제(non-maximum suppression)**: 각 픽셀에서 그래디언트 방향 $\phi$를
   따라 이웃 두 픽셀과 크기를 비교해, 국소 최대가 아니면 제거한다. 이로써 두꺼운
   경계가 1픽셀 폭 곡선이 된다.
4. **이력 임계(hysteresis thresholding)**: 두 임계값 $t_{high} > t_{low}$를 두고,
   $t_{high}$ 이상은 확정 엣지, $t_{low}$ 미만은 제거, 그 사이는 확정 엣지와
   연결된 경우에만 유지한다. 강한 증거에서 출발해 약한 증거를 따라가는 이 규칙이
   끊김과 잡음을 동시에 줄인다.

깨끗한 이진 마스크가 입력이므로 임계값 선택은 관대해도 되며, 이것이 1단계
딥러닝이 제공하는 이점이다.

**(b) Hough 직선 변환.** 엣지 픽셀들에서 직선을 찾는다. 직선을 원점 거리
$\rho$와 법선 각도 $\theta$로 매개화하면

$$
\rho = x \cos\theta + y \sin\theta
$$

이고, 엣지 픽셀 $(x, y)$ 하나는 $(\theta, \rho)$ 파라미터 공간에서 곡선 하나에
해당한다. 모든 엣지 픽셀이 자신이 지날 수 있는 직선들에 투표하면, 실제 직선이
있는 자리의 누적값이 커진다. 누적기(accumulator)의 국소 최대들이 검출된 직선이다.
확률적 변형(HoughLinesP)은 픽셀 일부만 표본화해 투표하고 선분(양 끝점)을
반환하므로 계산이 싸고 위치 정보가 풍부하다.

**(c) 4변 선택과 교점 계산.** 검출된 선분들을 사각형의 4개 변으로 정리한다.
선분의 방향각을 기준으로 두 개의 지배적 방향군(가로 계열/세로 계열)으로 나누고,
각 방향군 안에서 마스크 무게중심 기준 양쪽(위/아래 또는 좌/우)으로 분리해 4개
변을 얻는다. 각 변의 선분들을 대표 직선으로 적합한 뒤, 이웃한 두 변의 직선
$l_1: a_1 x + b_1 y + c_1 = 0$과 $l_2: a_2 x + b_2 y + c_2 = 0$의 교점을 선형
연립으로 푼다.

$$
\begin{pmatrix} a_1 & b_1 \\ a_2 & b_2 \end{pmatrix}
\begin{pmatrix} x \\ y \end{pmatrix}
= \begin{pmatrix} -c_1 \\ -c_2 \end{pmatrix}
$$

행렬식 $a_1 b_2 - a_2 b_1$이 0에 가까우면(두 직선이 평행) 교점이 정의되지 않으
므로 실패 처리한다. 네 교점을 TL $\to$ TR $\to$ BR $\to$ BL로 정렬하면 코너 초기 추정
$\hat{Y}^{(0)}$을 얻는다.

**(d) cornerSubPix 서브픽셀 정제.** 초기 코너를 원본 이미지의 그래디언트로
다듬는다. 원리는 다음 관찰이다: 진짜 코너점 $q$ 주변의 임의 픽셀 $p$에서, 이미지
그래디언트 $\nabla I(p)$는 (엣지 위에서는) 엣지에 수직이고 벡터 $q - p$는 엣지에
평행하므로 두 벡터는 직교한다. 균일 영역에서는 그래디언트가 0이라 자동으로
만족된다. 따라서 $q$는 다음을 최소화하는 점이다.

$$
q^* = \arg\min_q \sum_{p \in W}
\Big( \nabla I(p)^\top (q - p) \Big)^2
$$

$W$는 초기 코너 중심의 작은 창(예: $11 \times 11$)이다. 목적 함수를 $q$로
미분해 0으로 두면 선형 방정식이 된다.

$$
\Big( \sum_{p \in W} G_p \Big) q = \sum_{p \in W} G_p \, p, \qquad
G_p = \nabla I(p) \, \nabla I(p)^\top
$$

$2 \times 2$ 행렬 역산 한 번으로 $q$가 갱신되며, 창을 새 위치로 옮겨 수렴할
때까지 반복한다. 해는 픽셀 격자에 제약되지 않는 연속 좌표이므로 서브픽셀
정밀도를 얻는다. 마지막으로 픽셀 좌표를 이미지 크기로 나누어 [0, 1]로 정규화한다.

**실패 모드.** 연쇄의 각 단계가 실패 지점이다: (i) 마스크 공백/파편화, (ii) 직선
4개 미검출 또는 방향군 분리 실패, (iii) 인접 변 평행(교점 발산), (iv) 교점이
이미지 밖이거나 퇴화. 어느 단계든 실패하면 성공 플래그를 False로 반환한다.
비교적 싼 대체 경로로, 마스크 컨투어의 폴리곤 근사(seg 방식)로 후퇴하는 fallback을
둘 수 있다.

## 3. 결론

DL + Classical CV Hybrid는 강건성(딥러닝)과 정밀성(고전 CV)을 단계로 분리해 둘
다 취하는 방법론이다. 장점은 다음과 같다.

- **CPU 친화적**: 학습 파라미터가 경량 세그멘테이션에만 있고, 고전 CV 연쇄는
  결정적이며 계산이 싸다.
- **서브픽셀 정밀도**: cornerSubPix의 연속 좌표 해가 격자 제약 없는 정밀도를
  제공한다.
- **진단 용이성**: 마스크, 엣지, 직선, 교점의 중간 산출물이 모두 시각화 가능해
  실패 단계를 즉시 특정할 수 있다.

한계는 파이프라인 길이 자체이다. 단계가 많을수록 실패 모드의 조합이 늘고, 각
단계의 임계값/파라미터(Canny 임계, Hough 투표 수, 창 크기 등)가 도메인 튜닝을
요구한다. 학습으로 흡수되지 않는 이 수동 튜닝이 유지보수 비용이다.

이 방법론은 물체 경계가 직선이고 대비가 선명하며 CPU 배포 제약이 큰 조건에서
가장 적합하고, 종단간 학습 계열과 "학습을 어디까지 쓸 것인가"라는 설계 축의
대조를 제공한다.

## 부록 A. 데이터 인터페이스

- **preprocessor**: 표준 코너 $(N, 4, 2)$를 채운 볼록 사각형 마스크 $(N, 1, H, W)$로
  래스터화한다. preprocessor 내부에서 torch 배치 반평면(half-plane) 판정으로 생성하며
  (seg와 동일 방식), seg 방법론과 동일한 타깃 형식이다.
- **postprocessor**: raw 출력 $(N, 1, H, W)$에 sigmoid + 이진화를 적용한 뒤
  Canny $\to$ HoughLinesP $\to$ 4변 그룹화 $\to$ 교점 $\to$ cornerSubPix(2.4절)를 거쳐
  $(N, 4, 2)$를 만든다. cornerSubPix의 그래디언트 입력은 원본 그레이스케일
  이미지를 사용한다.
- **실패 모드**: 직선 미검출, 방향군 분리 실패, 교점 발산, 퇴화 판정
  (`is_invalid_corners`) 시 마스크 컨투어 폴리곤 근사(seg 방식)로 fallback을
  시도하고, 그마저 실패하면 성공 플래그를 False로 반환한다. Evaluator가
  SR(Success Rate)을 집계하고 정확도 메트릭은 성공 표본에만 적용한다 (README.md
  4절).

## 부록 B. 학습 전략

3단계 학습 전략을 세그멘테이션 단계에만 적용한다.

| 단계 | 데이터 | 목적 | 학습률 |
|---|---|---|---|
| 1. Pre-training | SmartDoc + MIDV-2020 (공개) | 일반적인 사각형 영역 분할 능력 학습 | 1e-3 |
| 2. Domain Adaptation | 합성 fringe 패턴 | 검사 도메인의 화면 특성에 적응 | 1e-4 |
| 3. Fine-tuning | 실측 데이터 (소량) | 최종 경계 정밀도 확보 | 1e-5 |

**후처리 파라미터 튜닝**: Canny 임계값, Hough 투표 임계/최소 선분 길이,
cornerSubPix 창 크기 등 비학습 파라미터는 학습과 별개로 검증셋에서 격자 탐색으로
정한다. 이 값들은 도메인(경계 대비, 해상도)에 민감하므로 단계 3의 실측 데이터
기준으로 최종 확정한다.

**인코더 초기화**: MobileNetV3 인코더는 ImageNet 사전학습 가중치에서 시작한다.

## 부록 C. 제약 적합성 분석

F1-F8 제약의 정의는 `README.md` 2절을 따른다.

| 제약 | 대응 |
|---|---|
| F1 (임의 볼록 사각형) | 4변 직선 교점 모델은 임의 볼록 사각형과 정확히 일치하는 표현. 근사 없음 |
| F2 (단일 객체, 50% 이상) | 변이 화면을 길게 가로지르므로 Hough 투표가 풍부하고 직선 검출이 안정적 |
| F3 (코너 경계 내부) | 교점과 cornerSubPix 창이 이미지 내부에 위치. 교점의 [0, 1] 범위 검증을 후처리에 포함 |
| F4 (실측 소량) | 학습 파라미터가 경량 세그멘테이션뿐이라 데이터 요구가 작고, 3단계 학습 전략(부록 B)으로 대응 |
| F5 (서브픽셀 정밀도) | cornerSubPix의 연속 좌표 해로 태생적 지원 (2.4절). heatmap과 함께 F5 상위 후보 |
| F6 (CPU 배포) | 본 프로젝트에서 F6 최적 후보. 분리 합성곱 기반 경량 모델 + 결정적 고전 CV 후처리 |
| F7 (조명/글레어 변동) | 광학 증강 적용. 고전 CV 연쇄가 원본이 아닌 정제된 마스크 위에서 동작하므로 조명 변동이 후처리에 직접 닿지 않음 |
| F8 (가림 없음) | 4변이 모두 온전히 보인다는 가정이 직선 그룹화/교점 계산 단순화의 근거 |
